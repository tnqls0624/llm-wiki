# 26 변경 이력과 용어집

Claude Code의 **용어집(glossary)**, **변경 이력(changelog)**, **주간 What's New 다이제스트(w13~w22)**, 그리고 팀 도입을 위한 **챔피언 킷 / 커뮤니케이션 킷**을 종합한 레퍼런스 노트다. 허브: [[Claude]].

- 용어집은 개념의 정의 사전이다. 각 항목이 실제로 다루어지는 상세 노트로 크로스링크를 걸었다.
- 변경 이력은 최근 약 3개월(2026-03 ~ 2026-06)의 굵직한 변화 위주로 압축했다. 빌드별 버그픽스 전체는 원본 changelog를 참조하라.
- 주간 다이제스트는 "이번 주 work 방식을 바꿀 만한 기능"만 골라낸 dev digest다. 사소한 개선·버그픽스는 changelog로 분리되어 있다.

> [!tip] 버전 확인
> `claude --version`으로 설치 버전을 확인한다. changelog는 GitHub의 `CHANGELOG.md`에서 생성된다. 자료 배포 전 [overview](https://code.claude.com/docs/en/overview)에서 버전별 세부사항을 검증하라(킷 문서 공통 주의).

## 용어집 (Glossary)

Claude Code 전용 용어 정의다. 토큰·temperature·RAG 같은 모델 레벨 개념은 [platform glossary](https://platform.claude.com/docs/en/about-claude/glossary)에 별도로 있다. 영문 용어는 그대로 두고 의미를 한국어로 풀었다.

### A

| 용어 | 정의 | 관련 노트 |
|------|------|-----------|
| **Agent teams** | 팀 리드가 조율하는 여러 독립 Claude Code 세션. 공유 task 리스트와 P2P 메시징을 가진다. subagent와 달리 각 teammate가 독립 컨텍스트 윈도우를 갖고 직접 대화 가능. 실험적 기능 — `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`로 활성화 | [[08 서브에이전트와 에이전트 팀]] |
| **Agentic coding** | AI가 파일을 읽고 명령을 실행하며 자율적으로 변경하는 동안 사람이 관찰·재지시·자리비움 하는 워크플로우. 텍스트만 답하는 챗봇과 대비됨 | [[Claude]] |
| **Agentic harness** | 언어 모델을 유능한 코딩 에이전트로 바꾸는 도구·컨텍스트 관리·실행 환경. Claude Code가 harness, Claude가 그 안의 model. 파일 접근·셸 실행·권한 게이팅·메모리 로딩·액션 체이닝 루프를 제공 | [[Claude]] |
| **Agentic loop** | 모든 task에서 도는 사이클: 컨텍스트 수집 → 액션 → 결과 검증 → 반복. 언제든 끼어들어 재지시 가능. hooks·skills·MCP 등 확장 지점이 이 루프의 특정 단계에 연결됨 | [[03 메모리와 컨텍스트]] |
| **Auto memory** | Claude가 사용자의 교정·선호를 바탕으로 스스로 작성하는 메모. git 저장소별로 `~/.claude/projects/` 아래 저장. 동일 저장소의 모든 worktree가 한 디렉터리를 공유. `MEMORY.md` 인덱스의 첫 200줄/25KB가 매 세션 시작 시 로드. CLAUDE.md(사람 작성)의 Claude-작성 대응물 | [[03 메모리와 컨텍스트]] |
| **Auto mode** | 별도 classifier 모델이 백그라운드에서 각 액션을 검토하는 권한 모드. 승인 프롬프트를 대체하고, 범위 확대·신뢰 불가 인프라·prompt injection을 차단. classifier는 tool 결과를 절대 보지 않아 주입된 지시에 영향받지 않음. research preview | [[05 권한]] |

### B

| 용어 | 정의 | 관련 노트 |
|------|------|-----------|
| **Bare mode** | `--bare` 시작 플래그. hooks·skills·plugins·MCP·auto memory·CLAUDE.md 자동 탐색을 건너뜀. 명시적으로 넘긴 플래그만 적용. CI/스크립트에서 머신 간 동일 동작이 필요할 때 권장 | [[13 자동화와 스케줄링]] |
| **Bundled skills** | Claude Code에 포함된 프롬프트 기반 플레이북(`/batch`, `/code-review`, `/debug`, `/loop` 등). 고정 로직을 실행하는 built-in 명령과 달리, 상세 프롬프트를 주고 Claude가 작업을 오케스트레이션 — 에이전트 생성·파일 읽기·코드베이스 적응 가능 | [[07 스킬]] |

### C

| 용어 | 정의 | 관련 노트 |
|------|------|-----------|
| **Channel** | 실행 중 세션에 이벤트를 푸시하는 MCP 서버. 터미널에서 떨어져 있어도 Claude가 반응 가능. 양방향 가능. Telegram·Discord·iMessage가 research preview에 포함 | [[10 MCP]] |
| **Checkpoint** | 보낸 프롬프트마다 생성되는 복원 지점. 매 편집 전 파일을 스냅샷. `Esc` 두 번 또는 `/rewind`로 코드·대화·둘 다를 이전으로 복원. 세션 로컬, git과 분리, Bash tool로 한 변경은 추적 안 함 | [[03 메모리와 컨텍스트]] |
| **`.claude` directory** | 프로젝트 범위 설정을 읽는 디렉터리: settings·hooks·skills·subagents·rules·auto memory. 프로젝트 루트의 `.claude/`, 사용자 기본값은 `~/.claude/` | [[04 설정]] |
| **CLAUDE.md** | 사람이 작성하는 지속 지시 마크다운. 매 세션 시작 시 system prompt 뒤 user 메시지로 로드. 프로젝트 컨벤션·아키텍처·"항상 X" 규칙을 둠. 프로젝트 루트 CLAUDE.md는 compaction을 견디고 이후 디스크에서 재읽기. `./CLAUDE.md`·`./.claude/CLAUDE.md`·`~/.claude/CLAUDE.md`·managed policy 배치 가능. 모든 발견 파일은 broad→specific 순으로 연결(concatenate) | [[03 메모리와 컨텍스트]] |
| **Command** | `/name`으로 호출하는 재사용 지시. built-in: `/clear`·`/model`·`/compact`. `.claude/commands/`에 직접 정의하거나 plugin에서 설치. 멀티스텝 패키징은 Skills 권장 | [[02 CLI 레퍼런스]] |
| **Compaction** | 컨텍스트 윈도우가 한계에 가까워질 때 대화를 자동 요약. 오래된 tool 출력 먼저 제거 후 대화 요약. 프로젝트 루트 CLAUDE.md와 auto memory는 살아남아 디스크에서 재로드. 대화로만 준 지시는 유실될 수 있음. `/compact`로 수동 트리거, `/compact focus on the API changes`처럼 포커스 지정 가능 | [[03 메모리와 컨텍스트]] |
| **Context window** | 세션의 작업 메모리. 대화 이력·파일 내용·명령 출력·CLAUDE.md·auto memory·로드된 skills·system 지시 보관. `/context`로 점유 확인 | [[03 메모리와 컨텍스트]] |

### H

| 용어 | 정의 | 관련 노트 |
|------|------|-----------|
| **Hook** | 라이프사이클 특정 지점(tool 실행 전, 파일 편집 후, 세션 시작 등)에서 자동 실행되는 사용자 정의 핸들러. 핸들러는 shell 명령·HTTP·MCP tool·LLM 프롬프트·subagent. 결정적(deterministic) — 모델 재량이 아닌 고정 지점에서 발화. 구성 3계층: **Hook event**(라이프사이클 지점) / **Matcher**(어느 이벤트를 발화시킬지 필터) / **Hook handler**(무엇이 실행될지) | [[09 훅]] |

### M

| **MCP Tool Search** | MCP tool 스키마를 필요 시점까지 지연하는 컨텍스트 절약 메커니즘. 시작 시 tool 이름만 로드, 사용 결정 시 전체 스키마를 on-demand로 가져옴. 유휴 MCP 서버의 컨텍스트 소비를 줄임 | [[10 MCP]] |

### P

| 용어 | 정의 | 관련 노트 |
|------|------|-----------|
| **Permission mode** | 세션의 기본 승인 동작. CLI에서 `Shift+Tab`으로 순환, VS Code·Desktop·claude.ai는 모드 선택기. 모드: `default`·`acceptEdits`·`plan`·`auto`·`dontAsk`·`bypassPermissions` | [[05 권한]] |
| **Permission rule** | tool 이름·인자 패턴 기반으로 invocation을 allow/ask/deny 하는 settings 엔트리. **deny→ask→allow** 순 평가, first match wins. permission mode 위에 얹는 세밀 제어 | [[05 권한]] |
| **Plan mode** | 소스 파일을 편집하지 않고 변경을 조사·제안하는 권한 모드. 읽기·검색·탐색 명령만 가능, 승인 후 변경. `/plan` 또는 `Shift+Tab`으로 진입 | [[05 권한]] |
| **Plugin** | skills·hooks·subagents·MCP 서버를 하나의 설치 단위로 묶은 번들. plugin skill은 `plugin-name:skill-name`으로 네임스페이스화. marketplace로 배포 | [[11 플러그인]] |
| **Project trust** | 설정 로드 전 디렉터리를 수락하는 다이얼로그. 프로젝트별로 저장(홈 디렉터리 예외 — 세션 한정, 매 실행 시 재프롬프트). marketplace plugin 자동설치·프로젝트 hook 실행을 게이팅 | [[18 보안과 샌드박스]] · [[04 설정]] |
| **Prompt injection** | 파일·웹페이지·tool 결과에 심긴 적대적 지시로 Claude를 의도치 않은 액션으로 유도. 방어: 권한 시스템·명령 blocklist·trust 검증. Auto mode는 tool 결과를 보지 않는 classifier + 서버측 probe를 추가 | [[18 보안과 샌드박스]] |

### S

| 용어 | 정의 | 관련 노트 |

| **Skill** | 지시·지식·워크플로우를 담은 `SKILL.md`. 관련 시 자동 로드 또는 `/skill-name`으로 직접 호출. Agent Skills 오픈 표준 준수. **custom command의 권장 후속** — `.claude/commands/deploy.md`와 `.claude/skills/deploy/SKILL.md`는 둘 다 `/deploy` 생성, 기존 command 파일도 계속 동작 | [[07 스킬]] |
| **Subagent** | 자체 컨텍스트 윈도우·custom system prompt·특정 tool 접근·독립 권한으로 도는 전문 AI. 위임 task 수행 후 main 대화에 요약 반환. 큰 탐색을 주 컨텍스트 밖으로 빼거나 병렬 조사에 사용. built-in: Explore·Plan·general-purpose. agent teams와 다름(각자 직접 대화 가능한 독립 세션) | [[08 서브에이전트와 에이전트 팀]] |
| **Surface** | Claude Code에 접근하는 곳: CLI·VS Code·JetBrains·Desktop·claude.ai. 모든 surface가 같은 엔진 공유 → CLAUDE.md·settings·skills가 동일 동작. Slack·Chrome 확장은 surface가 아닌 integration | [[14 IDE와 데스크톱]] · [[15 웹과 모바일]] |

1. **Week 1 — 채널 시드**: 채널 생성, Quickstart 핀, 본인 예시 2~3개(프롬프트 포함). 신호: 동료 반응 + 질문 1개.
2. **Week 2 — 리듬 시작**: 주간 show-and-tell 스레드, 모든 질문 공개 답변, custom skill/CLAUDE.md 스니펫 1개. 신호: 본인 외 누군가 예시 게시.
3. **Week 3 — 페어 & 통합**: 짧은 페어링 2~3회, 공통 Q&A를 핀 FAQ로. 신호: 반복 사용.
4. **Week 4 — 핸드오프**: 두 번째 챔피언 식별, 리드·관리자에게 요약 공유. 신호: 본인 외 사람이 질문에 답함.

### 흔한 우려 → 응답

| 우려 | 권장 응답 |
|------|-----------|
| "나 혼자가 더 빨라" | 일상 코드는 그렇다. 피하는 일(legacy·낯선 서비스·테스트 scaffold)에서 leverage 최대 |

| "또 다른 도구 배울 시간 없어" | 플랫폼이 아닌 터미널 명령. 첫 세션에 가치 없으면 접어도 됨 |

| 테크닉 | 적용법 |
|--------|--------|
| 올바른 context 제공 | `@file`·`@directory/`, 에러·로그 붙여넣기 — 정교한 프롬프트보다 효과적 |
| 편집 전 plan 리뷰 | `Shift+Tab`으로 plan mode |
| repo 가르치기 | `/init`으로 `CLAUDE.md` → 컨벤션·테스트 명령·회피 디렉터리 추가 |

| long task 알림 | Stop hook으로 데스크톱 알림 |
| 틀린 결과 복구 | 실패 테스트·스택 트레이스를 그대로 붙여넣기 |
| 편집 surgical 유지 | diff 요청 또는 "only change X" 명시 |

## 팀 도입 — 커뮤니케이션 킷 (Communications kit)

**관리자·엔지니어링 리드**가 조직에 롤아웃할 때 쓰는 복사-가능 출시 공지·드립 캠페인·FAQ 모음이다.

> [!note] 모두 draft copy
> 조직의 voice로 다시 쓰고, 예시 task를 실제 버그·모듈로 교체하고, `[bracketed placeholders]`를 채운 뒤 보내라. 도입을 이끄는 공지는 "우리 회사 사람이 쓴 것처럼" 읽히는 것이다.

### 출시 전 체크리스트

| 항목 | 왜 |
|------|-----|
| `#claude-code` 채널 생성·메시지에 링크 | 질문이 모일 한 곳 |
| 설치 명령을 환경 내 머신 1대에서 테스트 | proxy·방화벽 이슈를 사전 포착 |
| 보안·데이터 처리 링크 준비(Data usage 등) | "내 코드 어디로 가?"가 첫 질문 |
| 구체적 첫 task 1개(실제 버그·파일) | 일반 예시는 전환 안 됨 |
| 첫 48시간 채널 owner 지정 | 미응답 출시일 질문이 모멘텀을 죽임 |
| C-suite 스폰서 섭외 | exec 발신 출시가 admin 발신보다 첫 주 도입↑ |

### 표준 출시 공지 (핵심 요소)

- Claude Code 정의: 터미널에서 도는 AI 코딩 에이전트, 실제 코드베이스를 읽고 debugging·refactor·tests·PR을 end-to-end 처리. autocomplete도 챗 윈도우도 아님. 위험한 것 전엔 권한을 물음.
- 2분 설치 경로:
  ```bash
  curl -fsSL https://claude.ai/install.sh | bash
  cd <your-repo>
  claude
  ```
  이후 `/init` 한 번 → Claude가 빌드 명령·컨벤션이 담긴 `CLAUDE.md` 작성.
- 시도할 task 예시: "The test in [file] is flaky. Figure out why and fix it" / "Walk me through how [module] handles [X]" / "Look at my working diff and tell me what's risky before I push"
- "내 코드 어디로 가?": 터미널에서 도며 Anthropic API와 직접 통신, 제3자 서버 없음. 파일 편집·명령 실행 전 물음. Enterprise 계약 하에 코드·프롬프트를 모델 훈련에 사용 안 함.

변형: **Executive sponsor** 변형(exec 명의, "설치하고 실제 task 1개에 써봐" 단일 요청), **Pilot group** 변형(파일럿 코호트, `#claude-code-pilot`에 피드백), **Champion recruitment DM**(`#claude-code` 최활동 2~3명에게).

### 모델 선택 (드립 캠페인 핵심 표)

| Model | Best for |
|-------|----------|
| Opus | 대규모 refactor, 복잡한 debugging, 아키텍처 결정, high-stakes 변경 |
| Sonnet | 일상 기능·버그·테스트·문서·코드리뷰. **권장 기본값** |
| Haiku | 빠른 질문, 포맷팅, 기계적 편집, 빠른 반복 |

`/model`로 전환, 세션 중 변경 가능. → [[19 비용과 성능]].

### 드립 캠페인 팁 항목 (각 1~2개/주 게시)

각 팁은 hook → payoff → "try it now" 프롬프트 → docs 링크 패턴. 정해진 순서 없이 독립.

| 팁 | 핵심 + try-it-now | 관련 노트 |
|-----|-------------------|-----------|

| `/init`과 CLAUDE.md | repo당 한 번 `/init` → 빌드·아키텍처·컨벤션이 담긴 `CLAUDE.md`. 2화면 이하 유지 | [[03 메모리와 컨텍스트]] |
| `@`-references | `@` + 파일 경로로 context에 직접 pull. `@` 후 Tab으로 autocomplete | [[06 내장 도구 레퍼런스]] |
| Permission modes | `Shift+Tab`으로 `default`·`acceptEdits`·`plan` 순환. plan이 trust-builder | [[05 권한]] |
| Checkpointing/`/rewind` | `Esc` 두 번 또는 `/rewind`로 파일 변경 포함 롤백. 자동 체크포인트 | [[03 메모리와 컨텍스트]] |

| Screenshots/images | 스크린샷을 터미널에 드래그, `Ctrl+V` 붙여넣기(macOS도 Ctrl+V). "what's wrong here?" | [[21 브라우저와 컴퓨터 사용]] |

### FAQ 1줄 응답

| 질문 | 응답 |
|------|------|
| "VS Code에서 돼?" | 예. VS Code 확장·JetBrains plugin, 같은 기능 → [[14 IDE와 데스크톱]] |
| "먼저 설정할 게 있어?" | 아니. 설치 후 repo에서 `claude`, `/init` 한 번 |
| "내 코드 어디로 가?" | 터미널에서 돌며 추론용 context를 Anthropic API에 전송, 제3자 서버 없음. Enterprise 하 무훈련 |
| "전체 repo를 봐?" | 접근 권한 준 만큼. working dir 내 읽기는 프롬프트 없음, 편집·셸·외부는 권한 게이팅 → [[05 권한]] |
| "Copilot과 뭐가 달라?" | Copilot은 줄 autocomplete. Claude Code는 파일 읽기·명령 실행·멀티파일 편집하는 agent |
| "처음 뭘?" | 귀찮아 미룬 버그. "The test in [file] is flaky" |

### 프롬프트 템플릿 (시작용)

| Task | Prompt |
|------|--------|
| Fix a bug | "the tests in [file] are failing, figure out why and fix it" |
| Understand code | "walk me through how [module] works, then tell me where the entry point is" |
| Safe refactor | "refactor [module] to [goal], use plan mode so I can review first" |
| Write tests | "write tests for [file] that cover the edge cases around [scenario]" |
| Review before commit | "look at my working diff and tell me what looks risky" |
| Open a PR | "fix [issue], write a conventional commit, and open a PR with a summary" |
| Make a skill | "make me a /ship skill that runs tests and lint before commit" |
| Debug a stack trace | "here's the stack trace, find the root cause, don't just paper over it" |

> Claude Code는 자주 배포된다. 내부 배포 전 [overview](https://code.claude.com/docs/en/overview)에서 버전별 세부사항을 검증하라.
