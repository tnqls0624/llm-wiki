# 09 훅 (Hooks)

훅(Hook)은 Claude Code 세션 수명주기의 특정 시점에서 **자동으로 실행되는 사용자 정의 셸 명령, HTTP 엔드포인트, MCP 도구 호출, 또는 LLM 프롬프트**다. 모델이 "선택적으로" 무언가를 하도록 기대하는 대신, 훅은 특정 동작이 **항상 결정론적(deterministic)으로 일어나도록** 강제한다. 프로젝트 규칙 강제, 반복 작업 자동화(예: 포맷팅), 외부 도구 통합 등에 쓴다.

판단이 필요한 결정(규칙으로 딱 떨어지지 않는 경우)에는 [프롬프트 기반 훅](#프롬프트-기반-훅-prompt-hooks)이나 [에이전트 기반 훅](#에이전트-기반-훅-agent-hooks)을 쓸 수 있다. 다른 확장 메커니즘은 [[07 스킬]], [[08 서브에이전트와 에이전트 팀]], [[11 플러그인]]을 참고하라.

> [!tip] 두 문서의 관계
> 이 노트는 두 공식 문서를 합친 것이다. **hooks-guide**(가이드)는 빠른 시작과 대표 사용 패턴 중심이고, **hooks**(레퍼런스)는 전체 이벤트 스키마·JSON 입출력·종료 코드·고급 기능을 다룬다. 처음 설정한다면 가이드 섹션부터, 정확한 필드를 찾을 땐 레퍼런스 섹션을 보면 된다.

## 훅 수명주기 (Hook Lifecycle)

이벤트가 발생하고 matcher가 일치하면, Claude Code는 이벤트 컨텍스트를 JSON으로 훅 핸들러에 전달한다. command 훅은 stdin으로, HTTP 훅은 POST 요청 본문으로 받는다. 핸들러는 입력을 검사하고, 행동을 취하고, 선택적으로 결정(decision)을 반환할 수 있다.

이벤트는 발생 빈도(cadence)에 따라 세 종류로 나뉜다:
- **세션당 1회**: `SessionStart`, `SessionEnd`
- **턴당 1회**: `UserPromptSubmit`, `Stop`, `StopFailure`
- **에이전트 루프 내 도구 호출마다**: `PreToolUse`, `PostToolUse`

### 전체 이벤트 목록

| Event | 발생 시점 |
| :--- | :--- |
| `SessionStart` | 세션 시작 또는 재개 시 |
| `Setup` | `--init-only`, 또는 `-p` 모드에서 `--init`/`--maintenance` 시. CI/스크립트의 1회성 준비용 |
| `UserPromptSubmit` | 프롬프트 제출 시, Claude가 처리하기 전 |
| `UserPromptExpansion` | 사용자가 입력한 명령이 프롬프트로 확장될 때, Claude 도달 전. 확장 차단 가능 |
| `PreToolUse` | 도구 호출 실행 전. 차단 가능 |
| `PermissionRequest` | 권한 다이얼로그가 표시될 때 |
| `PermissionDenied` | auto 모드 분류기가 도구 호출을 거부할 때. `{retry: true}`로 재시도 허용 가능 |
| `PostToolUse` | 도구 호출 성공 후 |
| `PostToolUseFailure` | 도구 호출 실패 후 |
| `PostToolBatch` | 병렬 도구 호출 배치 전체가 끝난 후, 다음 모델 호출 전 |
| `Notification` | Claude Code가 알림을 보낼 때 |
| `MessageDisplay` | 어시스턴트 메시지 텍스트가 화면에 표시되는 동안 |
| `SubagentStart` | 서브에이전트가 생성될 때 |
| `SubagentStop` | 서브에이전트가 끝날 때 |
| `TaskCreated` | `TaskCreate`로 작업이 생성될 때 |
| `TaskCompleted` | 작업이 완료로 표시될 때 |
| `Stop` | Claude가 응답을 마칠 때 |
| `StopFailure` | API 에러로 턴이 끝날 때. 출력·종료코드 무시됨 |
| `TeammateIdle` | [에이전트 팀](#) 팀원이 유휴 상태로 들어가려 할 때 |
| `InstructionsLoaded` | `CLAUDE.md` 또는 `.claude/rules/*.md`가 컨텍스트에 로드될 때 |
| `ConfigChange` | 세션 중 설정 파일이 변경될 때 |
| `CwdChanged` | 작업 디렉터리 변경 시(예: `cd`). direnv 같은 도구와 함께 환경 관리에 유용 |
| `FileChanged` | 감시 중인 파일이 디스크에서 변경될 때. `matcher`가 감시 파일명 지정 |
| `WorktreeCreate` | `--worktree`/`isolation: "worktree"`로 worktree 생성 시. 기본 git 동작 대체 |
| `WorktreeRemove` | worktree 제거 시(세션 종료 또는 서브에이전트 완료 시) |
| `PreCompact` | 컨텍스트 압축 전 |
| `PostCompact` | 컨텍스트 압축 완료 후 |
| `Elicitation` | MCP 서버가 도구 호출 중 사용자 입력을 요청할 때 |
| `ElicitationResult` | 사용자가 MCP elicitation에 응답한 후, 서버로 응답 보내기 전 |
| `SessionEnd` | 세션 종료 시 |

### 훅이 해석(resolve)되는 과정

훅은 세 단계로 구성된다: **이벤트 → matcher 그룹(필터) → 핸들러**. 다음은 파괴적 셸 명령을 차단하는 `PreToolUse` 예시다. `matcher`가 Bash 도구로 좁히고, `if` 조건이 `rm *` 하위 명령으로 더 좁혀서, 두 필터가 모두 일치할 때만 `block-rm.sh`가 실행된다(프로세스 스폰 비용 절약).

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "if": "Bash(rm *)",
            "command": "${CLAUDE_PROJECT_DIR}/.claude/hooks/block-rm.sh",
            "args": []
          }
        ]
      }
    ]
  }
}
```

스크립트는 stdin에서 JSON을 읽고 명령을 추출해, `rm -rf`가 있으면 `permissionDecision: "deny"`를 반환한다:

```bash
#!/bin/bash
# .claude/hooks/block-rm.sh
COMMAND=$(jq -r '.tool_input.command')

if echo "$COMMAND" | grep -q 'rm -rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "Destructive command blocked by hook"
    }
  }'
else
  exit 0  # 결정 없음; 정상 권한 흐름 적용
fi
```

흐름: ①이벤트 발생(stdin으로 JSON 전달) → ②matcher가 `"Bash"` 일치 → ③`if` 조건 `"Bash(rm *)"` 일치 → ④핸들러 실행(`rm -rf` 발견 시 deny 출력) → ⑤Claude Code가 결정을 읽고 차단·이유 표시.

> [!note] 침묵은 승인이 아니다
> 종료 코드 0 + 출력 없음은 "보고할 결정 없음"을 뜻한다. `PreToolUse`에서 훅은 호출을 **거부할 수는 있지만**, 침묵한다고 **승인되는 것은 아니다**. 침묵 시 정상 [[05 권한]] 흐름이 그대로 적용된다.

## 구성 (Configuration)

훅은 JSON 설정 파일에 정의한다([[04 설정]]). 세 단계 중첩 구조다: ①응답할 [이벤트](#훅-수명주기-hook-lifecycle) 선택 → ②[matcher 그룹](#matcher-패턴) 추가(언제 발생할지 필터) → ③일치 시 실행할 [핸들러](#훅-핸들러-필드) 정의.

### 훅 위치 (Scope)

훅을 어디에 정의하느냐가 범위를 결정한다:

| 위치 | 범위 | 공유 가능 |
| :--- | :--- | :--- |
| `~/.claude/settings.json` | 모든 프로젝트 | 아니오 (로컬 머신) |
| `.claude/settings.json` | 단일 프로젝트 | 예 (레포에 커밋) |
| `.claude/settings.local.json` | 단일 프로젝트 | 아니오 (gitignore됨) |
| Managed policy settings | 조직 전체 | 예 (관리자 제어) |
| Plugin `hooks/hooks.json` | 플러그인 활성화 시 | 예 (플러그인에 번들) |
| Skill / agent frontmatter | 컴포넌트 활성 동안 | 예 (컴포넌트 파일에 정의) |

엔터프라이즈 관리자는 `allowManagedHooksOnly`로 사용자·프로젝트·플러그인 훅을 차단할 수 있다. managed `enabledPlugins`로 강제 활성화된 플러그인의 훅은 예외라, 관리자가 검증된 훅을 마켓플레이스로 배포 가능하다. 설정 파일 해석 순서는 [[04 설정]] 참고.

### matcher 패턴

`matcher` 필드는 훅이 언제 발생할지 필터링한다. 평가 방식은 포함된 문자에 따라 달라진다:

| matcher 값 | 평가 방식 | 예시 |
| :--- | :--- | :--- |
| `"*"`, `""`, 생략 | 모두 일치 | 모든 이벤트 발생 시 실행 |
| 문자·숫자·`_`·`\|`만 포함 | 정확한 문자열, 또는 `\|`로 구분된 정확한 문자열 목록 | `Bash`는 Bash만; `Edit\|Write`는 둘 중 하나 정확히 일치 |
| 그 외 문자 포함 | JavaScript 정규식 | `^Notebook`은 Notebook으로 시작하는 도구; `mcp__memory__.*`는 memory 서버의 모든 도구 |

이벤트별로 matcher가 필터링하는 필드가 다르다:

| 이벤트 | matcher가 거르는 대상 | 예시 값 |
| :--- | :--- | :--- |
| `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `PermissionRequest`, `PermissionDenied` | 도구 이름 | `Bash`, `Edit\|Write`, `mcp__.*` |
| `SessionStart` | 세션 시작 방식 | `startup`, `resume`, `clear`, `compact` |
| `Setup` | 트리거한 CLI 플래그 | `init`, `maintenance` |
| `SessionEnd` | 세션 종료 이유 | `clear`, `resume`, `logout`, `prompt_input_exit`, `bypass_permissions_disabled`, `other` |
| `Notification` | 알림 유형 | `permission_prompt`, `idle_prompt`, `auth_success`, `elicitation_dialog`, `elicitation_complete`, `elicitation_response` |
| `SubagentStart`/`SubagentStop` | 에이전트 유형 | `general-purpose`, `Explore`, `Plan`, 커스텀 이름 |
| `PreCompact`/`PostCompact` | 압축 트리거 | `manual`, `auto` |
| `ConfigChange` | 설정 소스 | `user_settings`, `project_settings`, `local_settings`, `policy_settings`, `skills` |
| `StopFailure` | 에러 유형 | `rate_limit`, `overloaded`, `authentication_failed`, `oauth_org_not_allowed`, `billing_error`, `invalid_request`, `model_not_found`, `server_error`, `max_output_tokens`, `unknown` |
| `InstructionsLoaded` | 로드 이유 | `session_start`, `nested_traversal`, `path_glob_match`, `include`, `compact` |
| `UserPromptExpansion` | 명령 이름 | 스킬/명령 이름 |
| `Elicitation`/`ElicitationResult` | MCP 서버 이름 | 설정된 MCP 서버 이름 |
| `FileChanged` | 감시할 리터럴 파일명 | `.envrc\|.env` |
| `UserPromptSubmit`, `PostToolBatch`, `Stop`, `TeammateIdle`, `TaskCreated`, `TaskCompleted`, `WorktreeCreate`, `WorktreeRemove`, `CwdChanged`, `MessageDisplay` | matcher 미지원 | 항상 발생 |

> matcher 미지원 이벤트에 `matcher` 필드를 추가하면 조용히 무시된다. `FileChanged`는 감시 목록 구성 시 위 규칙을 따르지 않는다([FileChanged](#filechanged) 참고).

#### MCP 도구 매칭

[[10 MCP]] 서버 도구는 일반 도구처럼 도구 이벤트에 나타나므로 동일하게 매칭한다. 이름 패턴은 `mcp__<server>__<tool>`:
- `mcp__memory__create_entities`, `mcp__filesystem__read_file`, `mcp__github__search_repositories`

서버의 모든 도구를 매칭하려면 `.*`를 붙여야 한다 (`.*` 필수 — `mcp__memory`는 문자/언더스코어만이라 정확 문자열로 취급되어 아무 도구와도 일치 안 함):
- `mcp__memory__.*` : memory 서버의 모든 도구
- `mcp__.*__write.*` : 어느 서버든 `write`로 시작하는 도구

### 훅 핸들러 필드

내부 `hooks` 배열의 각 객체가 핸들러다. 5가지 타입이 있다:

| 타입 | 동작 |
| :--- | :--- |
| `command` | 셸 명령 실행. stdin으로 JSON 입력, 종료코드·stdout으로 결과 통신 |
| `http` | JSON 입력을 URL에 POST. 응답 본문으로 결과 통신 |
| `mcp_tool` | 이미 연결된 MCP 서버의 도구 호출. 도구의 텍스트 출력을 command stdout처럼 취급 |
| `prompt` | Claude 모델에 단일 턴 평가 프롬프트 전송. yes/no JSON 결정 반환 |
| `agent` | 도구(Read, Grep, Glob 등)를 쓰는 서브에이전트 스폰. **실험적** |

#### 공통 필드

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `type` | yes | `"command"`, `"http"`, `"mcp_tool"`, `"prompt"`, `"agent"` 중 하나 |
| `if` | no | 권한 규칙 문법으로 언제 실행할지 필터 (예: `"Bash(git *)"`, `"Edit(*.ts)"`). 도구 이벤트에서만 평가됨. 다른 이벤트에선 `if` 설정 시 절대 실행 안 됨 |
| `timeout` | no | 취소까지 초. 기본값: command/http/mcp_tool 600, prompt 30, agent 60. `UserPromptSubmit`는 command/http/mcp_tool을 30으로, `MessageDisplay`는 10으로 낮춤 |
| `statusMessage` | no | 훅 실행 중 표시할 커스텀 스피너 메시지 |
| `once` | no | `true`면 세션당 1회 실행 후 제거. **스킬 frontmatter 훅에서만** 적용; 설정 파일·에이전트 frontmatter에선 무시 |

`if` 필드는 정확히 하나의 권한 규칙만 담는다. `&&`·`||`·목록 문법 없음 — 여러 조건은 핸들러를 따로 정의한다.

##### Bash `if` 매칭 표

Bash 패턴은 패턴 모양과 실행 명령에 따라 실행 여부가 달라진다. 선행 `VAR=value` 할당은 매칭 전 제거된다.

| `if` 패턴 | Bash 명령 | 실행? | 이유 |
| :--- | :--- | :--- | :--- |
| `Bash(git *)` | `FOO=bar git push` | yes | 선행 할당 제거됨; `git push` 일치 |
| `Bash(git *)` | `npm test && git push` | yes | 각 하위 명령 검사; `git push` 일치 |
| `Bash(rm *)` | `echo $(rm -rf /)` | yes | `$()`·백틱 내부 명령 검사; `rm -rf /` 일치 |
| `Bash(rm *)` | `echo $(date)` | no | `rm *` 일치 하위 명령 없음 |
| `Bash(git push *)` | `echo $(date)` | yes | 명령 이름 너머를 제약하는 패턴은 `$()`·백틱·`$VAR`에서 fail open |

> [!warning] `if`는 best-effort
> 명령 파싱 실패 시에도 fail open(패턴 무관하게 실행)한다. 강제 허용/거부를 **enforce**하려면 훅이 아니라 [[05 권한]] 시스템을 써라.

#### command 훅 추가 필드

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `command` | yes | 실행할 셸 명령. `args`와 함께 쓰면 직접 스폰할 실행파일 |
| `args` | no | 인자 목록. 있으면 `command`를 실행파일로 해석해 셸 없이 직접 스폰 |
| `async` | no | `true`면 블로킹 없이 백그라운드 실행 |
| `asyncRewake` | no | `true`면 백그라운드 실행 + 종료코드 2 시 Claude를 깨움. `async` 함의. stderr(없으면 stdout)를 시스템 리마인더로 표시 |
| `shell` | no | `"bash"`(기본) 또는 `"powershell"`. Windows에서 PowerShell로 실행. `args` 설정 시 무시 |

##### Exec form vs Shell form

- **Exec form** (`args` 있음): `command`를 PATH의 실행파일로 해석해 직접 스폰. **셸 없음** → 각 `args` 요소가 인용 없이 그대로 하나의 인자. 경로 플레이스홀더(`${CLAUDE_PLUGIN_ROOT}` 등)는 평문 문자열로 치환. 특수문자(`'`, `$`, 백틱)는 verbatim 통과. **경로 플레이스홀더를 쓰는 훅은 exec form을 선호하라.**
- **Shell form** (`args` 없음): `command` 문자열을 셸에 전달 (macOS/Linux는 `sh -c`, Windows는 Git Bash 또는 PowerShell). 셸이 토큰화·변수확장·파이프·`&&`·리다이렉트·glob 해석. **파이프나 `&&` 같은 셸 기능이 필요할 때 사용.**

```json
// Exec form: 인용 불필요
{ "type": "command", "command": "node", "args": ["${CLAUDE_PLUGIN_ROOT}/scripts/format.js", "--fix"] }
// Shell form: 공백/특수문자 대비 인용 필요
{ "type": "command", "command": "node \"${CLAUDE_PLUGIN_ROOT}\"/scripts/format.js --fix" }
```

> Windows의 exec form은 `command`가 실제 실행파일(`.exe`)이어야 한다. npm/npx/eslint의 `.cmd`/`.bat` 심은 스폰 불가 → `"command": "node", "args": ["...eslint.js"]` 패턴을 쓰거나 shell form 사용.

#### HTTP 훅 추가 필드

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `url` | yes | POST 보낼 URL |
| `headers` | no | 추가 HTTP 헤더. 값은 `$VAR_NAME`/`${VAR_NAME}` 환경변수 보간 지원 (단 `allowedEnvVars`에 나열된 것만) |
| `allowedEnvVars` | no | 헤더 값에 보간 가능한 환경변수 이름 목록. 보간 작동에 필수 |

JSON 입력을 `Content-Type: application/json`으로 POST 본문에 보낸다. command 훅과 에러 처리가 다르다: non-2xx·연결 실패·타임아웃은 모두 **비차단 에러**로 실행이 계속된다. 도구 호출을 차단하려면 2xx 응답 + `decision: "block"` 또는 `permissionDecision: "deny"`를 담은 JSON 본문을 반환해야 한다.

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "http",
        "url": "http://localhost:8080/hooks/pre-tool-use",
        "timeout": 30,
        "headers": { "Authorization": "Bearer $MY_TOKEN" },
        "allowedEnvVars": ["MY_TOKEN"]
      }]
    }]
  }
}
```

#### MCP 도구 훅 추가 필드

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `server` | yes | 설정된 MCP 서버 이름. 이미 연결돼 있어야 함(OAuth/연결 흐름 트리거 안 함) |
| `tool` | yes | 호출할 도구 이름 |
| `input` | no | 도구 인자. 문자열 값은 `${tool_input.file_path}` 같은 JSON 입력 치환 지원 |

도구의 텍스트 콘텐츠는 command stdout처럼 취급된다(유효 JSON이면 결정으로, 아니면 평문). 서버 미연결 또는 `isError: true` 반환 시 비차단 에러. `SessionStart`·`Setup`은 서버 연결 전 발생하므로 첫 실행 시 "not connected" 에러를 예상해야 한다.

#### prompt/agent 훅 추가 필드

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `prompt` | yes | 모델에 보낼 프롬프트. `$ARGUMENTS`로 훅 입력 JSON 주입. `\$1.00`로 리터럴 이스케이프 |
| `model` | no | 평가 모델. 기본은 빠른 모델 |

일치하는 모든 훅은 병렬 실행되며 동일 핸들러는 자동 중복제거(command는 명령문자열+args, HTTP는 URL 기준). 핸들러는 현재 디렉터리·Claude Code 환경에서 실행. 원격 웹 환경에선 `$CLAUDE_CODE_REMOTE="true"`가 설정됨.

### 경로로 스크립트 참조

작업 디렉터리와 무관하게 스크립트를 참조하는 플레이스홀더:
- `${CLAUDE_PROJECT_DIR}` : 프로젝트 루트 (stdio MCP 서버·플러그인 LSP 환경에도 설정됨)
- `${CLAUDE_PLUGIN_ROOT}` : 플러그인 설치 디렉터리 (업데이트마다 변경)
- `${CLAUDE_PLUGIN_DATA}` : 플러그인 [영속 데이터 디렉터리](#) (업데이트 시 보존되어야 할 의존성·상태용)

경로 플레이스홀더를 쓰는 훅은 exec form을 선호하라(셸 토큰화 없음 → 공백·특수문자 경로도 인용 불필요). shell form에선 각 플레이스홀더를 큰따옴표로 감싼다.

### 스킬과 에이전트의 훅

[[07 스킬]]과 [[08 서브에이전트와 에이전트 팀]]은 frontmatter에 직접 훅을 정의할 수 있다. 이 훅은 컴포넌트 수명에 스코프되어 해당 컴포넌트가 활성일 때만 실행되고 끝나면 정리된다. 모든 이벤트 지원. 서브에이전트의 경우 `Stop` 훅은 자동으로 `SubagentStop`으로 변환된다.

```yaml
---
name: secure-operations
description: Perform operations with security checks
hooks:
  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./scripts/security-check.sh"
---
```

### `/hooks` 메뉴

Claude Code에서 `/hooks`를 입력하면 **읽기 전용** 브라우저가 열린다. 모든 이벤트와 설정된 훅 개수를 보여주고, matcher·핸들러 세부를 드릴다운할 수 있다. 5가지 타입 모두 `[type]` 접두사와 소스 라벨로 표시된다:

| 소스 라벨 | 출처 |
| :--- | :--- |
| `User` | `~/.claude/settings.json` |
| `Project` | `.claude/settings.json` |
| `Local` | `.claude/settings.local.json` |
| `Plugin` | 플러그인 `hooks/hooks.json` |
| `Session` | 현재 세션 메모리 등록 |
| `Built-in` | Claude Code 내부 등록 |

메뉴는 읽기 전용 — 추가·수정·삭제는 설정 JSON을 직접 편집하거나 Claude에게 요청한다.

### 훅 비활성화·제거

- 제거: 설정 JSON에서 해당 항목 삭제
- 일시 전체 비활성화: 설정 파일에 `"disableAllHooks": true`. 개별 훅만 비활성화하는 방법은 없음
- `disableAllHooks`는 managed 계층을 존중 — 사용자/프로젝트/로컬에 설정해도 managed policy 훅은 못 끈다. managed 수준의 `disableAllHooks`만 managed 훅을 끌 수 있음

## 훅 입력과 출력 (Input & Output)

command 훅은 stdin으로 JSON을 받고 종료코드·stdout·stderr로 결과를 통신한다. HTTP 훅은 같은 JSON을 POST 본문으로 받고 HTTP 응답 본문으로 통신한다.

> [!note] /dev/tty 사용 불가
> v2.1.139부터 macOS/Linux의 command 훅은 controlling terminal 없는 세션에서 실행되어 `/dev/tty`에 직접 쓸 수 없다. 사용자에게 메시지를 표시하려면 JSON 출력의 [`systemMessage`](#json-출력-필드)를, 데스크톱 알림·창 제목·벨을 트리거하려면 [`terminalSequence`](#터미널-알림-emit)를 반환하라.

### 공통 입력 필드

| 필드 | 설명 |
| :--- | :--- |
| `session_id` | 현재 세션 식별자 |
| `transcript_path` | 대화 JSON 경로 |
| `cwd` | 훅 호출 시점 작업 디렉터리 |
| `permission_mode` | 현재 [[05 권한]] 모드: `"default"`, `"plan"`, `"acceptEdits"`, `"auto"`, `"dontAsk"`, `"bypassPermissions"`. 모든 이벤트에 있는 건 아님 |
| `effort` | `level` 필드를 가진 객체: `"low"`/`"medium"`/`"high"`/`"xhigh"`/`"max"`. 도구 사용 컨텍스트 이벤트에서 제공. `$CLAUDE_EFFORT` 환경변수로도 사용 가능 |
| `hook_event_name` | 발생한 이벤트 이름 |

`--agent` 또는 서브에이전트 내부 실행 시 추가: `agent_id`(서브에이전트 고유 ID), `agent_type`(에이전트 이름). `model` 필드는 `SessionStart` 훅에만 있다(`$CLAUDE_MODEL` 환경변수는 없음).

`PreToolUse` Bash 명령 훅의 stdin 예시:
```json
{
  "session_id": "abc123",
  "transcript_path": "/home/user/.claude/projects/.../transcript.jsonl",
  "cwd": "/home/user/my-project",
  "permission_mode": "default",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": { "command": "npm test" }
}
```

### 종료 코드 출력

| 종료 코드 | 의미 |
| :--- | :--- |
| **0** | 성공. stdout에서 JSON 출력 파싱(JSON은 종료코드 0에서만 처리). 대부분 이벤트는 stdout을 디버그 로그에만 기록 — 예외는 `UserPromptSubmit`, `UserPromptExpansion`, `SessionStart`로 stdout이 Claude의 컨텍스트로 추가됨 |
| **2** | 차단 에러. stdout·JSON 무시, stderr가 Claude에 에러 메시지로 전달. 효과는 이벤트별로 다름 |
| **그 외** | 비차단 에러. transcript에 `<hook name> hook error` + stderr 첫 줄 표시. 실행은 계속, 전체 stderr는 디버그 로그로 |

> [!warning] 정책 강제엔 반드시 exit 2
> 대부분 이벤트에서 **오직 종료코드 2만 차단**한다. Unix 관례상 실패 코드인 1조차 Claude Code는 비차단 에러로 취급하고 동작을 진행한다. 예외는 `WorktreeCreate` — 0이 아닌 모든 종료코드가 생성을 중단시킨다.

#### 이벤트별 종료코드 2 동작

| 이벤트 | 차단? | exit 2 시 |
| :--- | :--- | :--- |
| `PreToolUse` | Yes | 도구 호출 차단 |
| `PermissionRequest` | Yes | 권한 거부 |
| `UserPromptSubmit` | Yes | 프롬프트 처리 차단 + 프롬프트 삭제 |
| `UserPromptExpansion` | Yes | 확장 차단 |
| `Stop` | Yes | Claude 정지 방지, 대화 계속 |
| `SubagentStop` | Yes | 서브에이전트 정지 방지 |
| `TeammateIdle` | Yes | 팀원 유휴 방지(계속 작업) |
| `TaskCreated` | Yes | 작업 생성 롤백 |
| `TaskCompleted` | Yes | 작업 완료 표시 방지 |
| `ConfigChange` | Yes | 설정 변경 차단(`policy_settings` 제외) |
| `PostToolBatch` | Yes | 다음 모델 호출 전 에이전트 루프 정지 |
| `PreCompact` | Yes | 압축 차단 |
| `Elicitation` | Yes | elicitation 거부 |
| `ElicitationResult` | Yes | 응답 차단(decline으로 변경) |
| `WorktreeCreate` | Yes | non-zero 종료코드면 worktree 생성 실패 |
| `PostToolUse`, `PostToolUseFailure` | No | stderr를 Claude에 표시(도구 이미 실행됨) |
| `Notification`, `SubagentStart`, `SessionStart`, `Setup`, `SessionEnd`, `CwdChanged`, `FileChanged`, `PostCompact` | No | stderr를 사용자에게만 표시 |
| `StopFailure` | No | 출력·종료코드 무시 |
| `PermissionDenied` | No | 종료코드·stderr 무시(거부 이미 발생). JSON `retry: true` 사용 |
| `WorktreeRemove` | No | 실패는 디버그 모드에서만 로그 |
| `InstructionsLoaded` | No | 종료코드 무시 |
| `MessageDisplay` | No | 원본 텍스트 표시 |

### HTTP 응답 처리

HTTP 훅은 종료코드·stdout 대신 상태코드·본문을 쓴다:
- **2xx + 빈 본문**: 성공 (exit 0 + 출력 없음과 동일)
- **2xx + 평문 본문**: 성공, 텍스트가 컨텍스트로 추가
- **2xx + JSON 본문**: 성공, command JSON 출력 스키마로 파싱
- **non-2xx / 연결 실패·타임아웃**: 비차단 에러, 실행 계속

> HTTP 훅은 상태코드만으로 차단 신호를 못 보낸다. 차단하려면 2xx + 결정 필드를 담은 JSON 본문을 반환한다.

### JSON 출력 필드

종료코드 대신 exit 0 + stdout JSON으로 더 세밀한 제어가 가능하다. **한 훅당 한 방식만**(종료코드 단독 또는 exit 0 + JSON). exit 2 시 JSON은 무시된다. 훅 출력 문자열(`additionalContext`, `systemMessage`, 평문 stdout 포함)은 **10,000자 상한** — 초과 시 파일로 저장되고 미리보기+경로로 대체된다.

JSON은 세 종류 필드를 지원한다: **유니버설 필드**(모든 이벤트), **top-level `decision`/`reason`**(일부 이벤트), **`hookSpecificOutput`**(`hookEventName` 필수인 중첩 객체).

| 필드 | 기본 | 설명 |
| :--- | :--- | :--- |
| `continue` | `true` | `false`면 훅 후 Claude가 처리를 완전 중단. 이벤트별 결정 필드보다 우선 |
| `stopReason` | 없음 | `continue: false` 시 사용자에게 표시(Claude엔 안 보임) |
| `suppressOutput` | `false` | `true`면 transcript에서 stdout 숨김(디버그 로그엔 남음) |
| `systemMessage` | 없음 | 사용자에게 표시할 경고 메시지 |
| `terminalSequence` | 없음 | Claude Code가 대신 emit할 터미널 이스케이프 시퀀스. OSC `0`/`1`/`2`/`9`/`99`/`777`과 BEL로 제한 |

```json
{ "continue": false, "stopReason": "Build failed, fix errors before continuing" }
```

#### 터미널 알림 emit

`terminalSequence`(v2.1.141+)는 `/dev/tty`에 직접 쓰는 대신 Claude Code의 터미널 쓰기 경로로 이스케이프 시퀀스를 emit한다. tmux·screen·Windows에서도 race-free. 허용 시퀀스: OSC 0/1/2(창·아이콘 제목), OSC 9(iTerm2·ConEmu·Windows Terminal·WezTerm 알림, `9;4` taskbar progress 포함), OSC 99(Kitty), OSC 777(urxvt·Ghostty·Warp), bare BEL. 허용 목록 밖(CSI 커서·색상, OSC 8 하이퍼링크, OSC 52 클립보드, OSC 1337 등)은 거부되고 필드 무시.

```bash
#!/bin/bash
# Notification 훅: Claude Code가 주의를 요할 때 데스크톱 알림
input=$(cat)
title="Claude Code"
body=$(jq -r '.message // "Needs your attention"' <<<"$input")
seq=$(printf '\033]777;notify;%s;%s\007' "$title" "$body")
jq -nc --arg seq "$seq" '{terminalSequence: $seq}'
```

#### Claude에 컨텍스트 추가 (`additionalContext`)

`additionalContext` 필드는 훅의 문자열을 Claude 컨텍스트 윈도우에 주입한다. Claude Code가 system reminder로 감싸 훅 발생 지점에 삽입 — 채팅 메시지로는 안 보이지만 다음 모델 요청 때 읽힌다. `hookSpecificOutput` 안에 이벤트 이름과 함께 반환:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PostToolUse",
    "additionalContext": "This file is generated. Edit src/schema.ts and run `bun generate` instead."
  }
}
```

리마인더 삽입 위치는 이벤트별로 다르다: `SessionStart`/`Setup`/`SubagentStart`는 대화 시작 전, `UserPromptSubmit`/`UserPromptExpansion`은 제출 프롬프트 옆, `PreToolUse`/`PostToolUse`/`PostToolUseFailure`/`PostToolBatch`는 도구 결과 옆, `Stop`/`SubagentStop`은 턴 끝.

용도: 환경 상태(현재 브랜치·배포 타깃·기능 플래그), 조건부 프로젝트 규칙, 외부 데이터(이슈·CI 결과). **불변 지침은 [[03 메모리와 컨텍스트]]의 CLAUDE.md를 선호하라**(스크립트 실행 없이 로드됨). 텍스트는 명령형 시스템 지시가 아니라 사실 진술로 작성하라("배포 타깃은 production이다") — 명령형은 prompt-injection 방어를 자극해 Claude가 컨텍스트가 아닌 표시로 처리할 수 있다. 주입된 텍스트는 transcript에 저장되므로 `--continue`/`--resume` 시 재실행 없이 재생되어 타임스탬프·SHA가 stale해질 수 있다(`SessionStart`는 resume 시 재실행).

#### 결정 제어 (Decision Control) 요약

| 이벤트 | 결정 패턴 | 핵심 필드 |
| :--- | :--- | :--- |
| UserPromptSubmit, UserPromptExpansion, PostToolUse, PostToolUseFailure, PostToolBatch, Stop, SubagentStop, ConfigChange, PreCompact | top-level `decision` | `decision: "block"`, `reason`. Stop/SubagentStop은 `hookSpecificOutput.additionalContext`도 허용 |
| TeammateIdle, TaskCreated, TaskCompleted | 종료코드 또는 `continue: false` | exit 2 차단+stderr 피드백. `{"continue": false, "stopReason": "..."}`로 팀원 완전 정지 |
| PreToolUse | `hookSpecificOutput` | `permissionDecision`(allow/deny/ask/defer), `permissionDecisionReason` |
| PermissionRequest | `hookSpecificOutput` | `decision.behavior`(allow/deny) |
| PermissionDenied | `hookSpecificOutput` | `retry: true` |
| WorktreeCreate | path 반환 | command는 stdout에 경로 출력; HTTP는 `hookSpecificOutput.worktreePath` |
| Elicitation / ElicitationResult | `hookSpecificOutput` | `action`(accept/decline/cancel), `content` |
| MessageDisplay | `hookSpecificOutput` | `displayContent`(화면 표시만 교체) |
| SessionStart, Setup, SubagentStart | 컨텍스트만 | `additionalContext` 등. 차단·결정 제어 없음 |
| WorktreeRemove, Notification, SessionEnd, PostCompact, InstructionsLoaded, StopFailure, CwdChanged, FileChanged | 없음 | 로깅·정리 같은 부수효과용 |

## 주요 이벤트 상세 (레퍼런스)

### SessionStart

세션 시작/재개 시 실행. 개발 컨텍스트 로딩이나 환경변수 설정에 유용. **매 세션 실행되므로 빠르게 유지하라.** `command`·`mcp_tool` 타입만 지원. matcher: `startup`(새 세션), `resume`(`--resume`/`--continue`/`/resume`), `clear`(`/clear`), `compact`(자동·수동 압축).

입력에 `source`, `model`, 선택적 `agent_type`·`session_title`. 출력 전용 필드:

| 필드 | 설명 |
| :--- | :--- |
| `additionalContext` | 대화 시작 전 컨텍스트 추가 |
| `initialUserMessage` | 세션 첫 사용자 메시지로 사용. [비대화형 모드](#)(`-p`)에서 적용 |
| `sessionTitle` | 세션 제목 설정(`/rename` 동일 효과). `startup`/`resume`에만 적용 |
| `watchPaths` | [FileChanged](#filechanged) 이벤트로 감시할 절대경로 배열 |
| `reloadSkills` | `true`면 SessionStart 훅 완료 후 스킬·명령 디렉터리 재스캔(같은 세션에서 설치한 스킬 사용 가능) |

이 이벤트는 평문 stdout이 이미 Claude에 도달하므로, 컨텍스트만 로드하는 훅은 JSON 없이 stdout에 직접 출력해도 된다.

#### 환경변수 영속화 (`CLAUDE_ENV_FILE`)

`SessionStart`·`Setup`·`CwdChanged`·`FileChanged` 훅은 `CLAUDE_ENV_FILE` 환경변수에 접근한다. 여기에 `export` 문을 쓰면 이후 모든 Bash 명령에서 사용 가능. 다른 훅 변수 보존을 위해 append(`>>`) 사용:

```bash
#!/bin/bash
if [ -n "$CLAUDE_ENV_FILE" ]; then
  echo 'export NODE_ENV=production' >> "$CLAUDE_ENV_FILE"
  echo 'export PATH="$PATH:./node_modules/.bin"' >> "$CLAUDE_ENV_FILE"
fi
exit 0
```

### Setup

`--init-only`, 또는 `-p` 모드에서 `--init`/`--maintenance` 시에만 발생. **정상 시작에선 발생 안 함.** CI/스크립트에서 명시적으로 트리거하는 1회성 의존성 설치·정리용. matcher: `init`, `maintenance`. `command`·`mcp_tool` 타입만 지원. 차단 불가, `CLAUDE_ENV_FILE` 접근 가능.

### InstructionsLoaded

`CLAUDE.md` 또는 `.claude/rules/*.md` 로드 시 발생([[03 메모리와 컨텍스트]]). 세션 시작 시(eager) + 지연 로드 시(예: 하위 디렉터리 진입, `paths:` frontmatter 조건 매칭). 비동기 관찰용으로 실행되며 차단·결정 제어 없음. matcher는 `load_reason`. 입력: `file_path`, `memory_type`(User/Project/Local/Managed), `load_reason`, `globs`, `trigger_file_path`, `parent_file_path`. 감사 로깅·컴플라이언스 추적용.

### UserPromptSubmit

프롬프트 제출 시 Claude 처리 전 실행. 컨텍스트 추가, 검증, 차단 가능. **기본 타임아웃 30초**(command/http/mcp_tool) — 매 프롬프트마다 모델 처리를 블로킹하므로 멈춘 훅은 세션을 정지시킨다. 입력에 `prompt`.

exit 0에서 컨텍스트 추가 2가지: 평문 stdout(transcript에 훅 출력으로 표시) 또는 JSON `additionalContext`(더 은밀하게 추가). 차단 출력 필드:

| 필드 | 설명 |
| :--- | :--- |
| `decision` | `"block"`이면 프롬프트 처리 방지 + 컨텍스트에서 삭제 |
| `reason` | block 시 사용자에게 표시(컨텍스트엔 안 추가) |
| `additionalContext` | 제출 프롬프트와 함께 컨텍스트에 추가 |
| `sessionTitle` | 프롬프트 내용 기반 세션 제목 설정 |
| `suppressOriginalPrompt` | block 시 `true`면 차단 메시지에서 원본 프롬프트 생략 |

### UserPromptExpansion

사용자가 입력한 슬래시 명령이 프롬프트로 확장될 때 Claude 도달 전 실행. `PreToolUse`가 못 잡는 경로를 커버한다 — `Skill` 도구 `PreToolUse`는 Claude가 도구를 호출할 때만 발생하지만, `/skillname`을 직접 입력하면 우회된다. `UserPromptExpansion`은 그 직접 경로에서 발생. matcher: `command_name`. 입력: `expansion_type`(`slash_command`/`mcp_prompt`), `command_name`, `command_args`, `command_source`, `prompt`. 출력: `decision: "block"`(확장 방지), `reason`, `additionalContext`.

### MessageDisplay

어시스턴트 메시지가 화면에 스트리밍되는 동안 실행. 새로 완성된 줄 배치가 준비될 때마다 실행되어 교체 텍스트를 렌더링(긴 메시지는 여러 번, 짧은 건 한 번). **표시 전용** — transcript와 Claude가 보는 것은 원본 유지(Claude는 교체본을 못 봄). 마크다운 제거, Agent SDK 앱 표시 변환, API 키·내부 호스트명 redaction에 사용. **기본 타임아웃 10초**, 빠르게 유지하라(실패·타임아웃 시 원본 표시). matcher 미지원. 비대화형 실행(Agent SDK·`claude -p`)에선 메시지당 1회 실행되며 전체 텍스트를 담음(`index: 0`, `final: true`).

입력: `turn_id`, `message_id`, `index`(배치 0기반 인덱스), `final`(마지막 배치 시 `true`), `delta`(이전 배치 이후 새 줄). 출력: `displayContent`(delta를 화면에서 교체).

```bash
# 마크다운 제거 예시
jq '{hookSpecificOutput: {hookEventName: "MessageDisplay", displayContent: (.delta | gsub("\\*\\*"; "") | gsub("`"; ""))}}'
```

### PreToolUse

Claude가 도구 파라미터를 만든 후, 도구 호출 처리 전 실행. 도구 이름으로 매칭: `Bash`, `Edit`, `Write`, `Read`, `Glob`, `Grep`, `Agent`, `WebFetch`, `WebSearch`, `AskUserQuestion`, `ExitPlanMode` 및 MCP 도구. 입력에 `tool_name`, `tool_input`, `tool_use_id`. `tool_input` 스키마는 도구별로 다르다(내장 도구 상세는 [[06 내장 도구 레퍼런스]]):

- **Bash**: `command`, `description`, `timeout`(ms), `run_in_background`
- **Write**: `file_path`, `content`
- **Edit**: `file_path`, `old_string`, `new_string`, `replace_all`
- **Read**: `file_path`, `offset`, `limit`
- **Glob**: `pattern`, `path`
- **Grep**: `pattern`, `path`, `glob`, `output_mode`, `-i`, `multiline`
- **WebFetch**: `url`, `prompt`
- **WebSearch**: `query`, `allowed_domains`, `blocked_domains`
- **Agent**: `prompt`, `description`, `subagent_type`, `model` ([[08 서브에이전트와 에이전트 팀]])
- **AskUserQuestion**: `questions`(배열), `answers`(프로그래밍 응답용)
- **ExitPlanMode**: `plan`, `planFilePath`, `allowedPrompts` (plan 모드 — [[05 권한]])

> `PostToolUse`의 Agent `tool_response`는 서브에이전트 최종 텍스트 + 사용량 텔레메트리(`totalTokens`, `usage`, `totalDurationMs`, `totalToolUseCount` 등)를 담는다. 훅에서 서브에이전트별 비용 기록 가능([[19 비용과 성능]]).

#### PreToolUse 결정 제어

다른 훅과 달리 `hookSpecificOutput` 안에 결정을 반환 → 4가지 결과 + 입력 수정.

| 필드 | 설명 |
| :--- | :--- |
| `permissionDecision` | `"allow"`(권한 프롬프트 스킵), `"deny"`(차단), `"ask"`(확인 프롬프트), `"defer"`(나중에 재개) |
| `permissionDecisionReason` | allow/ask는 사용자에게, deny는 Claude에게, defer는 무시 |
| `updatedInput` | 실행 전 도구 입력 파라미터 수정. **전체 입력 객체 교체**(변경 안 한 필드도 포함) |
| `additionalContext` | 도구 결과와 함께 컨텍스트 추가. defer 시 무시 |

여러 PreToolUse 훅이 다른 결정 반환 시 우선순위: **`deny` > `defer` > `ask` > `allow`**. [deny/ask 규칙](#)은 훅 반환과 무관하게 평가됨(훅 allow가 deny 규칙을 무시 못 함).

> [!note] 권한 모드와의 관계
> PreToolUse 훅은 모든 권한 모드 체크 **이전**에 발생한다. `permissionDecision: "deny"`는 `bypassPermissions` 모드나 `--dangerously-skip-permissions`에서도 차단한다 — 사용자가 모드를 바꿔도 우회 못 하는 정책 강제 가능. 반대로 `"allow"`는 설정의 deny 규칙을 우회 못 함. **훅은 제약을 강화할 순 있어도 권한 규칙 너머로 완화할 순 없다.**

> [!warning] deprecated
> PreToolUse는 과거 top-level `decision`/`reason`을 썼으나 deprecated. `hookSpecificOutput.permissionDecision`/`permissionDecisionReason`를 쓰라. 구값 `"approve"`/`"block"`은 `"allow"`/`"deny"`로 매핑됨.

#### defer로 도구 호출 보류

`"defer"`(v2.1.89+)는 `claude -p`를 서브프로세스로 돌리고 JSON 출력을 읽는 통합(Agent SDK 앱·커스텀 UI)용이다. 비대화형 모드(`-p`)에서만 동작(대화형에선 경고 후 무시). 라운드트립: ①Claude가 도구 호출 → ②훅이 `"defer"` 반환, 프로세스가 `stop_reason: "tool_deferred"`로 종료, 호출 보존 → ③호출 프로세스가 `deferred_tool_use`를 읽어 자체 UI로 입력 수집 → ④`claude -p --resume <session-id>` 실행 → ⑤훅이 `updatedInput`에 답을 담아 `"allow"` 반환. 타임아웃·재시도 제한 없음. **단일 도구 호출일 때만 동작**(배치면 무시).

### PermissionRequest

권한 다이얼로그가 표시될 때 실행. matcher는 PreToolUse와 동일(도구 이름). 입력은 `tool_name`·`tool_input`(단 `tool_use_id` 없음) + 선택적 `permission_suggestions`("always allow" 옵션 배열). 출력 `decision` 객체:

| 필드 | 설명 |
| :--- | :--- |
| `behavior` | `"allow"`(허가) / `"deny"`(거부). deny/ask 규칙은 여전히 평가됨 |
| `updatedInput` | allow 전용: 입력 수정(전체 교체). deny/ask 규칙 재평가 |
| `updatedPermissions` | allow 전용: [권한 업데이트 항목](#권한-업데이트-항목) 배열 |
| `message` | deny 전용: Claude에게 거부 이유 |
| `interrupt` | deny 전용: `true`면 Claude 정지 |

#### 권한 업데이트 항목

`updatedPermissions`와 `permission_suggestions`가 공유하는 항목. 각 항목은 `type`과 `destination`을 가짐.

| `type` | 필드 | 효과 |
| :--- | :--- | :--- |
| `addRules` | `rules`, `behavior`, `destination` | 규칙 추가. `rules`는 `{toolName, ruleContent?}` 배열 |
| `replaceRules` | `rules`, `behavior`, `destination` | 해당 behavior 규칙 전체 교체 |
| `removeRules` | `rules`, `behavior`, `destination` | 일치 규칙 제거 |
| `setMode` | `mode`, `destination` | 권한 모드 변경(`default`/`auto`/`acceptEdits`/`dontAsk`/`bypassPermissions`/`plan`) |
| `addDirectories` | `directories`, `destination` | 작업 디렉터리 추가 |
| `removeDirectories` | `directories`, `destination` | 작업 디렉터리 제거 |

`destination`: `session`(메모리 전용), `localSettings`, `projectSettings`, `userSettings`. `setMode bypassPermissions`는 세션이 bypass 모드로 시작됐을 때만 효과(아니면 no-op), `defaultMode`로 영속화 안 됨.

### PostToolUse

도구 성공 직후 실행. matcher는 도구 이름. 입력에 `tool_input` + `tool_response`(결과) + `duration_ms`(권한 프롬프트·PreToolUse 훅 시간 제외). 출력 필드:

| 필드 | 설명 |
| :--- | :--- |
| `decision` | `"block"`이면 도구 결과 옆에 `reason` 추가(Claude는 원본도 봄) |
| `reason` | block 시 Claude에게 표시 |
| `additionalContext` | 도구 결과와 함께 컨텍스트 추가 |
| `updatedToolOutput` | Claude에게 보내기 전 도구 출력 교체(도구의 출력 shape에 맞아야 함) |
| `updatedMCPToolOutput` | MCP 도구 전용. `updatedToolOutput` 선호 |

> [!warning] PostToolUse는 되돌릴 수 없음
> `updatedToolOutput`은 Claude가 보는 것만 바꾼다. 훅 발생 시점엔 도구가 **이미 실행됨** — 파일 쓰기·명령·네트워크 요청은 이미 적용됐다. OpenTelemetry 스팬 등 텔레메트리도 원본을 캡처한다. 실행 전 차단·수정은 [PreToolUse](#pretooluse)를 써라. 내장 도구는 구조화 객체를 반환(`Bash`는 `stdout`/`stderr`/`interrupted`/`isImage`) — 스키마 불일치 값은 무시되고 원본 사용.

### PostToolUseFailure

도구 실행 실패 시 실행(에러 throw 또는 실패 결과). matcher는 도구 이름. 입력에 `error`(설명 문자열), `is_interrupt`(사용자 인터럽트 여부), `duration_ms`. 출력은 `additionalContext`만(차단 불가).

### PostToolBatch

배치 내 모든 도구 호출이 끝난 후, 다음 모델 요청 전 **정확히 1회** 실행(matcher 없음). `PostToolUse`는 도구마다(병렬 시 동시) 발생하지만 `PostToolBatch`는 배치 전체로 1회 → 실행된 도구 집합에 의존하는 컨텍스트 주입에 적합. 입력 `tool_calls` 배열(각 호출의 `tool_name`/`tool_input`/`tool_use_id`/`tool_response`). 

> `tool_response`는 모델이 보는 직렬화된 `tool_result` 콘텐츠(PostToolUse의 구조화 Output과 다름). `Read`는 줄번호 접두 텍스트. 응답이 클 수 있으니 필요한 필드만 파싱.

출력 `additionalContext`(다음 모델 호출 전 1회 주입). `decision: "block"`/`continue: false`로 에이전트 루프 정지.

### PermissionDenied

[auto 모드](#) 분류기가 도구 호출을 거부할 때만 실행(수동 다이얼로그 거부·PreToolUse 차단·deny 규칙 매칭 시엔 발생 안 함). matcher는 도구 이름. 입력에 `reason`(분류기 설명). 출력 `hookSpecificOutput.retry: true`면 모델에게 재시도 가능 메시지 추가(거부 자체는 안 뒤집힘).

### Notification

Claude Code 알림 전송 시 실행. matcher: `permission_prompt`, `idle_prompt`, `auth_success`, `elicitation_dialog`, `elicitation_complete`, `elicitation_response`. 입력에 `message`, 선택적 `title`, `notification_type`. 차단·수정 불가(외부 서비스 전달 등 부수효과용). `systemMessage` 등 공통 JSON 출력 적용.

### SubagentStart / SubagentStop

`SubagentStart`: 서브에이전트 생성 시. matcher는 에이전트 유형. 입력 `agent_id`, `agent_type`. 차단 불가, `additionalContext`로 서브에이전트에 컨텍스트 주입 가능.

`SubagentStop`: 서브에이전트 응답 완료 시. matcher는 에이전트 유형. 입력에 `stop_hook_active`, `agent_id`, `agent_type`, `agent_transcript_path`(서브에이전트 자체 transcript), `last_assistant_message`(최종 응답 텍스트). [Stop 결정 제어](#stop)와 동일 포맷 — `decision: "block"` + `reason`은 서브에이전트를 계속 작동시킴. 부모 세션에 컨텍스트 주입은 `Agent` 도구의 `PostToolUse` 훅을 써라.

### TaskCreated / TaskCompleted

`TaskCreated`: `TaskCreate` 도구로 작업 생성 시. 명명 규칙 강제·설명 필수화 등에 사용. exit 2면 작업 미생성 + stderr 피드백. `{"continue": false, "stopReason": "..."}`면 팀원 완전 정지. 입력 `task_id`, `task_subject`, 선택적 `task_description`/`teammate_name`/`team_name`.

```bash
# 티켓 번호 형식 강제 예시
#!/bin/bash
INPUT=$(cat)
TASK_SUBJECT=$(echo "$INPUT" | jq -r '.task_subject')
if [[ ! "$TASK_SUBJECT" =~ ^\[TICKET-[0-9]+\] ]]; then
  echo "Task subject must start with a ticket number, e.g. '[TICKET-123] Add feature'" >&2
  exit 2
fi
exit 0
```

`TaskCompleted`: 작업 완료 표시 시(TaskUpdate로 명시 완료, 또는 팀원이 in-progress 작업 남기고 턴 종료). 완료 기준(테스트·린트 통과) 강제에 사용. exit 2면 완료 미표시 + 피드백. 위 두 이벤트 모두 matcher 미지원. [[08 서브에이전트와 에이전트 팀]] 참고.

### Stop

메인 Claude Code 에이전트가 응답을 마칠 때 실행. **사용자 인터럽트 시엔 실행 안 됨**, API 에러는 [StopFailure](#stopfailure). 입력에 `stop_hook_active`(stop 훅 결과로 이미 계속 중이면 `true` — 무한 차단 방지용으로 확인 필수), `last_assistant_message`, `background_tasks`(진행 중 작업 배열), `session_crons`(세션 스코프 예약 wakeup 배열, v2.1.145+).

> `/goal` 명령은 세션 스코프 prompt 기반 Stop 훅의 내장 단축이다.

#### Stop 결정 제어

| 필드 | 설명 |
| :--- | :--- |
| `decision` | `"block"`이면 Claude 정지 방지 |
| `reason` | block 시 필수. Claude에게 계속할 이유 |
| `hookSpecificOutput.additionalContext` | 비에러 피드백. 대화 계속(같은 루프 보호 적용) but transcript엔 hook feedback으로 표시(hook error 아님) |

> [!warning] 8회 차단 cap
> Claude Code는 Stop 훅이 진전 없이 **연속 8회 차단**하면 오버라이드하고 턴을 끝낸다. 훅은 `stop_hook_active`를 파싱해 이미 트리거됐으면 일찍 exit 0 해야 한다. 8회 이상 필요하면 `CLAUDE_CODE_STOP_HOOK_BLOCK_CAP` 환경변수로 cap 상향.

### StopFailure

API 에러로 턴이 끝날 때 [Stop](#stop) 대신 실행. 출력·종료코드 무시. matcher는 `error` 유형(`rate_limit`, `overloaded`, `authentication_failed`, `oauth_org_not_allowed`, `billing_error`, `invalid_request`, `model_not_found`, `server_error`, `max_output_tokens`, `unknown`). 입력 `error`, `error_details`, `last_assistant_message`(여기선 API 에러 문자열). 결정 제어 없음(알림·로깅용). [[25 트러블슈팅]] 관련.

### TeammateIdle

에이전트 팀 팀원이 턴 후 유휴 상태로 들어가려 할 때 실행. 린트 통과·출력 파일 확인 같은 품질 게이트에 사용. exit 2면 팀원이 stderr 피드백 받고 계속 작업. `{"continue": false, "stopReason": "..."}`면 완전 정지. matcher 미지원. 입력 `teammate_name`, `team_name`. [[08 서브에이전트와 에이전트 팀]].

### ConfigChange

세션 중 설정 파일 변경 시 실행. 설정·managed policy·스킬 파일 변경에 발생. matcher: `user_settings`, `project_settings`, `local_settings`, `policy_settings`, `skills`. 입력 `source`, 선택적 `file_path`. exit 2 또는 `decision: "block"`으로 변경 차단(새 설정이 실행 세션에 미적용). **`policy_settings`는 차단 불가**(엔터프라이즈 설정 항상 적용 — 훅은 발생하나 차단 결정 무시). 감사 로깅 예시:

```json
{
  "hooks": {
    "ConfigChange": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "jq -c '{timestamp: now | todate, source: .source, file: .file_path}' >> ~/claude-config-audit.log"
      }]
    }]
  }
}
```

### CwdChanged

작업 디렉터리 변경 시(예: `cd`) 실행. 환경변수 리로드·툴체인 활성화·setup 스크립트에 사용. [FileChanged](#filechanged)와 짝지어 direnv 같은 per-디렉터리 환경 관리. `CLAUDE_ENV_FILE` 접근 가능, matcher 미지원. 입력 `old_cwd`, `new_cwd`. 출력 `watchPaths`(FileChanged 동적 감시 목록 설정 — 빈 배열은 동적 목록 비움). 차단 불가.

### FileChanged

감시 파일이 디스크에서 변경될 때 실행. `matcher`가 **두 역할**: ①감시 목록 구성(`|`로 분리해 각 세그먼트를 작업 디렉터리의 리터럴 파일명으로 등록 — 정규식 무용), ②변경 시 어느 훅 그룹 실행할지 필터(표준 matcher 규칙). `CLAUDE_ENV_FILE` 접근 가능. 입력 `file_path`, `event`(`change`/`add`/`unlink`). 출력 `watchPaths`(동적 감시 목록 갱신). 차단 불가.

### WorktreeCreate / WorktreeRemove

`claude --worktree` 또는 서브에이전트 `isolation: "worktree"` 시 Claude Code가 `git worktree`로 격리 작업 복사본을 만든다. `WorktreeCreate` 훅 설정 시 **기본 git 동작을 대체** → SVN·Perforce·Mercurial 등 다른 VCS 사용 가능. 기본 동작을 완전 대체하므로 `.worktreeinclude`가 처리 안 됨(`.env` 등 복사는 훅 스크립트에서). 훅은 생성된 worktree 디렉터리의 **절대경로를 반환해야 함**(command는 stdout, HTTP는 `hookSpecificOutput.worktreePath`). 입력 `name`(슬러그 식별자). non-zero 종료코드면 생성 실패.

```json
// SVN worktree 예시
{
  "hooks": {
    "WorktreeCreate": [{
      "hooks": [{
        "type": "command",
        "command": "bash -c 'NAME=$(jq -r .name); DIR=\"$HOME/.claude/worktrees/$NAME\"; svn checkout https://svn.example.com/repo/trunk \"$DIR\" >&2 && echo \"$DIR\"'"
      }]
    }]
  }
}
```

`WorktreeRemove`: worktree 제거 시(`--worktree` 세션 종료 + 제거 선택, 또는 서브에이전트 완료). git 기반은 자동 정리, non-git VCS는 짝 훅으로 정리해야 함. 입력 `worktree_path`. 차단 불가, 실패는 디버그 모드에서만 로그.

### PreCompact / PostCompact

`PreCompact`: 압축 직전. matcher: `manual`(`/compact`), `auto`(컨텍스트 윈도우 가득 참). exit 2 또는 `decision: "block"`으로 차단. 입력 `trigger`, `custom_instructions`(manual은 `/compact` 인자, auto는 빈 값). 자동 압축 차단은 트리거 시점에 따라 효과가 다름 — 선제적 트리거면 압축 스킵하고 계속, 컨텍스트 한계 에러 복구용이면 에러가 표면화되어 요청 실패.

`PostCompact`: 압축 완료 후. matcher 동일. 입력 `trigger`, `compact_summary`(생성된 요약). 결정 제어 없음. [[03 메모리와 컨텍스트]] 압축 관련.

### SessionEnd

세션 종료 시 실행. 정리·통계 로깅·상태 저장에 사용. matcher는 `reason`: `clear`, `resume`, `logout`, `prompt_input_exit`, `bypass_permissions_disabled`, `other`. 차단 불가. **기본 타임아웃 1.5초**(세션 종료·`/clear`·`/resume` 전환). 더 필요하면 per-hook `timeout` 설정(예산이 최고 per-hook 타임아웃까지 자동 상향, 최대 60초; 플러그인 훅 타임아웃은 예산 상향 안 함). 명시적 오버라이드는 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS`(ms):

```bash
CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS=5000 claude
```

### Elicitation / ElicitationResult

`Elicitation`: MCP 서버가 작업 중 사용자 입력 요청 시 실행([[10 MCP]]). 기본은 대화형 다이얼로그, 훅이 가로채 프로그래밍 응답 가능(다이얼로그 스킵). matcher는 MCP 서버 이름. 입력 `mcp_server_name`, `message`, 선택적 `mode`(`form`/`url`), `url`, `elicitation_id`, `requested_schema`. 출력 `action`(accept/decline/cancel), `content`(accept 시 폼 값). exit 2면 거부.

`ElicitationResult`: 사용자가 elicitation에 응답한 후 서버로 보내기 전 실행. 응답 관찰·수정·차단 가능. 입력 `action`, `content` 등. 출력으로 사용자 응답 오버라이드. exit 2면 차단(action을 decline으로).

## 프롬프트 기반 훅 (prompt hooks)

판단이 필요한 결정에는 `type: "prompt"` 훅을 쓴다. 셸 명령 대신 프롬프트 + 훅 입력을 Claude 모델(기본 Haiku)에 보내 yes/no JSON 결정을 받는다. `model` 필드로 더 강한 모델 지정 가능.

```json
{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "prompt",
        "prompt": "Evaluate if Claude should stop: $ARGUMENTS. Check if all tasks are complete."
      }]
    }]
  }
}
```

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `type` | yes | `"prompt"` |
| `prompt` | yes | LLM에 보낼 텍스트. `$ARGUMENTS`로 훅 입력 JSON 주입(없으면 입력이 프롬프트에 append) |
| `model` | no | 평가 모델. 기본은 빠른 모델 |
| `timeout` | no | 초. 기본 30 |
| `continueOnBlock` | no | `ok: false` 시 이유를 Claude에 피드백하고 턴 계속(정지 대신). 기본 `false` |

응답 스키마: `{ "ok": true|false, "reason": "..." }`. `ok: false` 시 동작은 이벤트별로 다르다:
- `Stop`/`SubagentStop`: reason을 다음 지시로 피드백, 턴 계속
- `PreToolUse`: 도구 거부 + reason을 도구 에러로 (command 훅 `permissionDecision: "deny"`와 동일)
- `PostToolUse`: 기본은 턴 종료 + 경고 라인. `continueOnBlock: true`면 피드백하고 계속
- `PostToolBatch`/`UserPromptSubmit`/`UserPromptExpansion`: 턴 종료 + 경고 라인(`continue` 무관)
- `PostToolUseFailure`/`TaskCreated`/`TaskCompleted`: reason을 도구 에러로 (PreToolUse 유사)
- `TeammateIdle`: 기본은 팀원 정지. `continueOnBlock: true`면 계속
- `PermissionRequest`/`PermissionDenied`: `ok: false` 무효 — command 훅을 써야 함

### 이벤트별 지원 타입

**5가지 모두**(`command`/`http`/`mcp_tool`/`prompt`/`agent`): `PermissionDenied`, `PermissionRequest`, `PostToolBatch`, `PostToolUse`, `PostToolUseFailure`, `PreToolUse`, `Stop`, `SubagentStop`, `TaskCompleted`, `TaskCreated`, `TeammateIdle`, `UserPromptExpansion`, `UserPromptSubmit`.

**`command`/`http`/`mcp_tool`만**(prompt·agent 미지원): `ConfigChange`, `CwdChanged`, `Elicitation`, `ElicitationResult`, `FileChanged`, `InstructionsLoaded`, `Notification`, `PostCompact`, `PreCompact`, `SessionEnd`, `StopFailure`, `SubagentStart`, `WorktreeCreate`, `WorktreeRemove`.

**`command`/`mcp_tool`만**: `SessionStart`, `Setup`(http·prompt·agent 미지원).

## 에이전트 기반 훅 (agent hooks)

> [!warning] 실험적
> agent 훅은 실험적이며 동작·구성이 바뀔 수 있다. 프로덕션 워크플로에는 command 훅을 선호하라.

`type: "agent"`는 prompt 훅과 유사하나 **다중 턴 도구 접근**을 가진다. 단일 LLM 호출 대신 서브에이전트를 스폰해 Read·Grep·Glob 등으로 코드베이스를 조사하고 조건을 검증한 뒤 결정을 반환한다. prompt 훅과 같은 이벤트 지원. 최대 50턴, 기본 타임아웃 60초. 응답 스키마는 prompt 훅과 동일(`{ "ok": true }` / `{ "ok": false, "reason": "..." }`).

```json
{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "agent",
        "prompt": "Verify that all unit tests pass. Run the test suite and check the results. $ARGUMENTS",
        "timeout": 120
      }]
    }]
  }
}
```

훅 입력 데이터만으로 결정 가능하면 prompt 훅, 실제 코드베이스 상태 검증이 필요하면 agent 훅을 쓴다.

## 백그라운드 실행 (async hooks)

기본적으로 훅은 완료까지 Claude 실행을 블로킹한다. 배포·테스트·외부 API 등 긴 작업은 `"async": true`로 백그라운드 실행하면서 Claude가 계속 작업한다. **async 훅은 차단·제어 불가** — `decision`/`permissionDecision`/`continue` 필드는 무효(대상 동작이 이미 완료됨). `type: "command"`만 지원.

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "${CLAUDE_PROJECT_DIR}/.claude/hooks/run-tests-async.sh",
        "args": [],
        "async": true,
        "timeout": 300
      }]
    }]
  }
}
```

백그라운드 프로세스 종료 후 `additionalContext` JSON을 생성했으면 **다음 대화 턴**에 컨텍스트로 전달된다(`systemMessage`는 사용자에게). 완료 알림은 기본 억제 — `Ctrl+O` 또는 `--verbose`로 표시. async 훅 출력은 세션이 유휴면 다음 사용자 상호작용까지 대기(예외: `asyncRewake` 훅이 exit 2면 유휴여도 즉시 Claude를 깨움). 동일 async 훅 다중 발생 시 중복제거 없음.

## 여러 훅 결과 결합

같은 이벤트에 여러 훅이 일치하면 모두 병렬 실행되어 각각 완료된 뒤 결과를 병합한다. 한 훅의 `deny`가 형제 훅 실행을 막지 못한다 — 한 훅의 deny로 다른 훅의 부수효과를 억제하려 하지 마라. `PreToolUse` 권한 결정은 가장 제한적인 답이 이긴다(`deny` > `ask` > `allow`). `additionalContext` 텍스트는 모든 훅에서 보존되어 함께 Claude에 전달된다.

```json
// Bash 로깅 + rm -rf 차단 (둘 다 실행, deny가 이김)
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [
        { "type": "command", "command": "jq -r .tool_input.command >> ~/.claude/bash.log" },
        { "type": "command", "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/block-rm-rf.sh" }
      ]
    }]
  }
}
```

## 대표 사용 패턴 (가이드)

### 첫 훅 설정 (데스크톱 알림)

`~/.claude/settings.json`에 `Notification` 훅 추가 → `/hooks`로 확인 → 권한 필요한 작업 시키고 터미널 전환 시 알림 수신. 기존 `hooks` 키가 있으면 이벤트 키를 형제로 추가(전체 객체 교체 X).

| 플랫폼 | 명령 |
| :--- | :--- |
| macOS | `osascript -e 'display notification "Claude Code needs your attention" with title "Claude Code"'` |
| Linux | `notify-send 'Claude Code' 'Claude Code needs your attention'` |
| Windows | `powershell.exe -Command "[System.Reflection.Assembly]::LoadWithPartialName('System.Windows.Forms'); [System.Windows.Forms.MessageBox]::Show('...', 'Claude Code')"` |

> macOS에서 알림이 안 뜨면 `osascript -e 'display notification "test"'`를 한 번 실행 후 System Settings > Notifications에서 Script Editor의 Allow Notifications를 켜라.

### 편집 후 자동 포맷

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{ "type": "command", "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write" }]
    }]
  }
}
```

> 이 페이지의 Bash 예시는 `jq`로 JSON 파싱한다. 설치: `brew install jq`(macOS), `apt-get install jq`(Debian/Ubuntu).

### 보호 파일 편집 차단

`.env`·`package-lock.json`·`.git/` 등 수정 방지. 스크립트가 경로를 패턴과 대조해 exit 2로 차단:

```bash
#!/bin/bash
# .claude/hooks/protect-files.sh
INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty')
PROTECTED_PATTERNS=(".env" "package-lock.json" ".git/")
for pattern in "${PROTECTED_PATTERNS[@]}"; do
  if [[ "$FILE_PATH" == *"$pattern"* ]]; then
    echo "Blocked: $FILE_PATH matches protected pattern '$pattern'" >&2
    exit 2
  fi
done
exit 0
```

`chmod +x` 후 `PreToolUse` matcher `Edit|Write`로 등록. 이 vault의 `protect-sources` 훅이 같은 원리다(`sources/` 보호).

### 압축 후 컨텍스트 재주입

`SessionStart` matcher `compact`로 압축 후 핵심 컨텍스트 재주입(stdout이 컨텍스트로 추가됨). `echo` 대신 `git log --oneline -5` 같은 동적 출력 가능.

### 권한 프롬프트 자동 승인

`PermissionRequest` matcher를 좁게 잡아 특정 도구만 자동 승인. 예: `ExitPlanMode` 자동 승인:

```json
{
  "hooks": {
    "PermissionRequest": [{
      "matcher": "ExitPlanMode",
      "hooks": [{
        "type": "command",
        "command": "echo '{\"hookSpecificOutput\": {\"hookEventName\": \"PermissionRequest\", \"decision\": {\"behavior\": \"allow\"}}}'"
      }]
    }]
  }
}
```

> [!warning] matcher를 최대한 좁게
> `.*`나 빈 matcher는 파일 쓰기·셸 명령 포함 **모든** 권한 프롬프트를 자동 승인한다.

`setMode`로 세션 모드 전환도 가능:
```json
{ "hookSpecificOutput": { "hookEventName": "PermissionRequest",
  "decision": { "behavior": "allow",
    "updatedPermissions": [{ "type": "setMode", "mode": "acceptEdits", "destination": "session" }] } } }
```

### direnv 환경 리로드

`SessionStart`(시작 디렉터리 변수 로드) + `CwdChanged`(디렉터리 변경마다 리로드)를 짝지어 `CLAUDE_ENV_FILE`에 `direnv export bash > "$CLAUDE_ENV_FILE"` 작성. 각 디렉터리에서 `direnv allow` 한 번 실행 필요. devbox/nix는 `devbox shellenv`로 대체.

## 보안 고려사항

> [!warning] 시스템 전체 권한
> command 훅은 **사용자의 전체 권한**으로 셸 명령을 실행한다. 사용자 계정이 접근 가능한 모든 파일을 수정·삭제·접근할 수 있다. 설정에 추가하기 전 모든 훅 명령을 검토·테스트하라.

베스트 프랙티스([[18 보안과 샌드박스]] 참고):
- **입력 검증·살균**: 입력 데이터를 맹신하지 마라
- **셸 변수 항상 인용**: `"$VAR"` (not `$VAR`)
- **path traversal 차단**: 파일 경로의 `..` 검사
- **절대경로 사용**: exec form은 `${CLAUDE_PROJECT_DIR}`(인용 불필요), shell form은 큰따옴표
- **민감 파일 스킵**: `.env`, `.git/`, 키 등

## Windows PowerShell

Windows에서 command 훅에 `"shell": "powershell"`을 설정하면 PowerShell로 실행된다. 훅이 PowerShell을 직접 스폰하므로 `CLAUDE_CODE_USE_POWERSHELL_TOOL` 설정 여부와 무관. `pwsh.exe`(7+) 자동 감지, `powershell.exe`(5.1) 폴백.

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{ "type": "command", "shell": "powershell", "command": "Write-Host 'File written'" }]
    }]
  }
}
```

## 디버깅

훅 실행 세부(일치한 훅, 종료코드, stdout/stderr)는 디버그 로그에 기록된다. `claude --debug-file <path>`로 알려진 위치에 쓰거나, `claude --debug` 후 `~/.claude/debug/<session-id>.txt`에서 읽는다(`--debug`는 터미널 출력 안 함). 더 세밀한 매칭 정보는 `CLAUDE_CODE_DEBUG_LOG_LEVEL=verbose`.

```text
[DEBUG] Executing hooks for PostToolUse:Write
[DEBUG] Found 1 hook commands to execute
[DEBUG] Executing hook command: <Your command> with timeout 600000ms
[DEBUG] Hook command completed with status 0: <Your stdout>
```

transcript 뷰(`Ctrl+O`)는 발생한 각 훅의 한 줄 요약을 보여준다(성공은 침묵, 차단 에러는 stderr, 비차단 에러는 `<hook name> hook error`).

## 한계와 트러블슈팅

### 주요 한계
- command 훅은 stdout·stderr·종료코드로만 통신 — `/` 명령이나 도구 호출을 트리거할 수 없다. `additionalContext`는 system reminder로 주입됨.
- 타임아웃은 타입별: command/http/mcp_tool 10분(`UserPromptSubmit` 30초, `MessageDisplay` 10초), prompt 30초, agent 60초. per-hook `timeout`(초)로 오버라이드.
- `PostToolUse`는 도구가 이미 실행되어 되돌릴 수 없다.
- `PermissionRequest`는 비대화형 모드(`-p`)에서 발생 안 함 → 자동 권한 결정엔 `PreToolUse`를 쓰라.
- `Stop`은 Claude가 응답을 마칠 때마다 발생(작업 완료 시점 아님), 사용자 인터럽트엔 발생 안 함.
- 여러 PreToolUse 훅이 `updatedInput`으로 인자를 재작성하면 마지막 완료가 이김(병렬이라 순서 비결정). 같은 도구 입력 수정은 하나만.

### 흔한 문제 ([[25 트러블슈팅]])

| 증상 | 해결 |
| :--- | :--- |
| 훅이 안 뜸 | `/hooks`로 확인. matcher가 도구 이름과 정확히 일치하는지(대소문자 구분), 올바른 이벤트인지. `-p`에서 `PermissionRequest` → `PreToolUse`로 |
| `hook error` 메시지 | 스크립트 수동 테스트: `echo '{"tool_name":"Bash","tool_input":{"command":"ls"}}' \| ./my-hook.sh; echo $?`. "command not found"면 절대경로/`${CLAUDE_PROJECT_DIR}` 또는 `"args": []`로 exec form. "jq: command not found"면 jq 설치. 안 돌면 `chmod +x` |
| `/hooks`에 안 보임 | 파일 워처가 놓쳤으면 세션 재시작. JSON 유효성 확인(trailing comma·주석 불가). 위치 확인 |
| Stop 훅 8회 cap | `stop_hook_active`를 파싱해 `true`면 exit 0. 더 필요하면 `CLAUDE_CODE_STOP_HOOK_BLOCK_CAP` |
| JSON 검증 실패 | 셸 프로파일의 무조건 `echo`가 JSON 앞에 붙음. `if [[ $- == *i* ]]; then echo ...; fi`로 대화형 셸에서만 출력하게 감싸기 |

## 이 vault에서의 응용

이 LLM Wiki(`obsidian_sync`)는 실제로 훅을 적극 활용한다:
- **`protect-sources`** (`PreToolUse`): `sources/` 디렉터리의 기존 파일에 대한 Write/Edit/MultiEdit + 파괴적 Bash(rm/mv/sed -i/redirect)를 차단 — 위 [보호 파일 차단](#보호-파일-편집-차단) 패턴의 best-effort 구현(OS 샌드박스 아님).
- **`wiki-lint-check.py`** (`PostToolUse`): 단일 파일 편집 시 frontmatter 필수 필드 검사.
- **`session-context`** (`SessionStart` 류): 안정적 컨텍스트 블록(hot.md + index 발췌) 주입 — 프롬프트 캐시 히트를 위해 블록을 안정·상단 유지([[03 메모리와 컨텍스트]], [[19 비용과 성능]]).

`.claude/`는 portable framework 패키지(훅 포함)이고 `wiki/`+`sources/`는 그 위의 데이터다.
