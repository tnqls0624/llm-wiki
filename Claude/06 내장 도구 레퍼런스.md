# 06 내장 도구 레퍼런스

Claude Code가 코드베이스를 이해하고 수정할 때 사용하는 **내장 도구(built-in tools)**와, 세션 안에서 Claude Code를 제어하는 **슬래시 커맨드(slash commands)** 전체 레퍼런스다. 도구 이름 문자열은 [[05 권한]]의 권한 규칙, [[08 서브에이전트와 에이전트 팀]]의 도구 목록, [[09 훅]]의 matcher에서 그대로 쓰이는 식별자이므로 정확한 표기가 중요하다. 허브: [[Claude]].

> [!info] 두 페이지의 관계
> - **도구(tools)**: Claude가 자율적으로 호출하는 실행 단위. 권한·서브에이전트·훅 설정에서 이름으로 참조한다.
> - **커맨드(commands)**: 사람이 `/`로 직접 호출하는 제어 명령. 일부 커맨드는 내부적으로 도구나 번들 스킬/워크플로우로 동작한다.

### 권한 규칙·훅으로 도구 설정하기

도구 이름을 직접 참조하는 위치는 다음과 같다. 모두 같은 규칙 포맷 `ToolName(specifier)`를 받는다.

- 설정의 [`permissions.allow`/`permissions.deny`](https://code.claude.com/docs/en/settings) 및 `/permissions` 인터페이스 ([[05 권한]])
- CLI 플래그 `--allowedTools` / `--disallowedTools` ([[02 CLI 레퍼런스]])
- Agent SDK의 `allowedTools` / `disallowedTools` ([[23 Agent SDK — 핵심 기능]])
- 서브에이전트 frontmatter의 `tools` / `disallowedTools` ([[08 서브에이전트와 에이전트 팀]])
- 스킬 frontmatter의 `allowed-tools` ([[07 스킬]])
- 훅의 `if` 조건 ([[09 훅]])

specifier는 도구마다 다르며 여러 도구가 포맷을 공유한다.

| Rule format | Applies to | Details |
| :--- | :--- | :--- |
| `Bash(npm run *)` | Bash, Monitor | 명령 패턴 매칭 |
| `PowerShell(Get-ChildItem *)` | PowerShell | 명령 패턴 매칭 |
| `Read(~/secrets/**)` | Read, Grep, Glob, LSP | 경로 패턴 매칭 |
| `Edit(/src/**)` | Edit, Write, NotebookEdit | 경로 패턴 매칭 |
| `Skill(deploy *)` | Skill | 스킬 이름 매칭 |
| `Agent(Explore)` | Agent | 서브에이전트 타입 매칭 |
| `WebFetch(domain:example.com)` | WebFetch | 도메인 매칭 |
| `WebSearch` | WebSearch | specifier 없음. 도구 전체를 allow/deny |

주의사항:
- 여기 없는 도구(`ExitPlanMode`, `ShareOnboardingGuide` 등)는 specifier 없이 **이름만** 받는다.
- `Edit(...)` allow 규칙은 같은 경로의 읽기 접근도 함께 부여한다. 별도 `Read(...)`가 불필요.
- 훅 `matcher` 필드는 괄호 규칙이 아니라 **이름만** 쓴다. 훅이 `tool_input`에 넘기는 필드명은 [[09 훅]]의 PreToolUse 입력 레퍼런스 참조.

### Agent 도구 동작

Agent 도구는 별도 컨텍스트 윈도우에서 서브에이전트를 생성한다. 서브에이전트는 작업을 자율 수행한 뒤 **단일 텍스트 결과**만 부모 대화에 반환한다. 부모는 중간 도구 호출/출력을 보지 못한다. 실행 턴 수 상한은 서브에이전트 정의의 `maxTurns`로 둔다.

fork 모드 활성 시 같은 Agent 도구가 [forked 서브에이전트](https://code.claude.com/docs/en/sub-agents#fork-the-current-conversation)도 띄운다. fork는 새 출발이 아니라 부모 대화 전체를 상속하고 항상 백그라운드로 돌며, 권한 프롬프트는 여전히 터미널에 뜬다.

서브에이전트의 도구 접근은 `tools`/`disallowedTools` 조합으로 결정된다.

| 설정 | 결과 |
| :--- | :--- |
| 둘 다 미설정 | 부모의 모든 도구 상속 |
| `tools`만 | 나열된 도구만 |
| `disallowedTools`만 | 나열된 것 제외한 부모 도구 전부 |
| 둘 다 | `disallowedTools` 우선(양쪽에 있으면 제거) |

- **Foreground 서브에이전트**: 각 도구 호출 시점에 메인 대화와 동일한 권한 프롬프트를 보여준다.
- **Background 서브에이전트**: 프롬프트 없음. 이미 부여된 권한으로 실행하고, 프롬프트가 필요한 호출은 자동 거부 후 그 도구 없이 계속 진행한다.

### Bash 도구 동작

각 명령을 별도 프로세스에서 실행한다. 영속성 동작:

- 메인 세션에서 `cd` 실행 시 새 작업 디렉터리가 이후 Bash 명령으로 이어진다 — 단, 프로젝트 디렉터리나 `--add-dir`/`/add-dir`/`additionalDirectories`로 추가한 디렉터리 안에 머무를 때만.
  - 범위 밖으로 `cd` 하면 프로젝트 디렉터리로 리셋되고 결과에 `Shell cwd was reset to <dir>`가 붙는다.
  - carry-over를 끄려면 `CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR=1`.
- **환경변수는 영속하지 않는다.** 한 명령의 `export`는 다음 명령에서 못 쓴다.
- 셸 시작 파일(`~/.zshrc`/`~/.bashrc`/`~/.profile`)의 alias·함수는 세션 시작 시 캡처되어 모든 Bash 명령에 적용된다.

환경변수를 영속시키려면 실행 전 [`CLAUDE_ENV_FILE`](https://code.claude.com/docs/en/env-vars)을 셸 스크립트로 지정하거나 SessionStart 훅으로 채운다 ([[04 설정]], [[09 훅]]).

명령당 두 한계:

| 한계 | 기본값 | 상한 | 오버라이드 |
| :--- | :--- | :--- | :--- |
| Timeout | 2분 | 10분 | `timeout` 파라미터 / `BASH_DEFAULT_TIMEOUT_MS`, `BASH_MAX_TIMEOUT_MS` |
| Output length | 30,000자 | 150,000자 | `BASH_MAX_OUTPUT_LENGTH` |

출력이 한도 초과 시 전체를 세션 디렉터리 파일에 저장하고 경로+앞부분 프리뷰만 제공한다. dev 서버·watch 빌드 등 장기 프로세스는 `run_in_background: true`로 백그라운드 작업화하고 `/tasks`로 나열·중단한다.

### Edit 도구 동작

`old_string`을 `new_string`으로 **정확히** 치환(regex/fuzzy 없음). 적용에 3가지 검사 통과 필요:

1. **Read-before-edit**: 현재 대화에서 파일을 읽었어야 하고, 그 후 디스크에서 변경되지 않았어야 함. 가장 먼저 검사.
2. **Match**: `old_string`이 작성한 그대로 등장해야 함. 공백/들여쓰기 한 글자 차이도 실패.
3. **Uniqueness**: `old_string`이 정확히 한 번만 등장해야 함. 여러 번이면 주변 컨텍스트를 더 붙이거나 `replace_all: true`.

Bash로 파일을 보는 것도 read-before-edit를 충족한다 — 단 단일 파일에 파이프/리다이렉트 없는 `cat`, `head`, `tail`, `sed -n 'X,Yp'`, `grep`, `egrep`, `fgrep` 일 때만.

> [!warning] read-before-edit 목록 ≠ deny 규칙 인식 명령
> Read/Edit deny 규칙은 `cat`/`head`/`tail`/`sed`/`grep` 같이 인식되는 파일 명령에도 적용되지만, Python/Node 스크립트처럼 간접적으로 파일을 여는 임의 서브프로세스에는 적용되지 않는다. 또 `egrep`/`fgrep`은 read-before-edit엔 포함되나 Read deny 규칙 검사 대상은 아니다. 모든 프로세스를 OS 수준에서 강제하려면 [[18 보안과 샌드박스]]의 샌드박스를 켠다.

### Glob 도구 동작

이름 패턴으로 파일 검색. 표준 glob 구문 + `**` 재귀 지원.

| 패턴 | 매칭 |
| :--- | :--- |
| `**/*.js` | 모든 깊이의 `.js` |
| `src/**/*.ts` | `src/` 하위 모든 `.ts` |
| `*.{json,yaml}` | 현재 디렉터리의 `.json`·`.yaml` |

결과는 수정 시각 정렬, 최대 100개. 캡 도달 시 truncation 플래그가 보여 패턴을 좁힐 수 있다. Glob은 기본적으로 `.gitignore`를 무시하지 않는다(= gitignore된 파일도 찾음). Grep과 반대. `.gitignore`를 존중시키려면 `CLAUDE_CODE_GLOB_NO_IGNORE=false`.

### Grep 도구 동작

[ripgrep](https://github.com/BurntSushi/ripgrep) 기반이라 POSIX grep이 아닌 ripgrep regex 구문을 쓴다. metacharacter는 이스케이프 필요(예: Go의 `interface{}` → `interface\{\}`).

출력 모드 3가지:

| 모드 | 반환 |
| :--- | :--- |
| `files_with_matches` | 파일 경로만 (기본) |
| `content` | 매칭 라인 + 파일·라인번호 |
| `count` | 파일별 매칭 수 |

- `glob` 파라미터(`**/*.tsx`)나 `type` 파라미터(`py`, `rust`)로 범위 한정.
- 기본은 단일 라인 내 매칭. `multiline: true`로 라인 경계 넘김.
- `.gitignore`를 **존중**(gitignore된 파일 건너뜀). 직접 경로를 넘기면 검색 가능.

### LSP 도구 동작

실행 중인 언어 서버에서 코드 인텔리전스를 받는다. 매 파일 편집 후 타입 에러/경고를 자동 보고해 별도 빌드 없이 수정 가능. 직접 호출로 정의 이동, 모든 참조 찾기, 위치별 타입 정보, 파일 내 심볼 목록, 워크스페이스 전역 심볼 검색, 인터페이스 구현 찾기, 호출 계층 추적 가능. 언어용 [code intelligence 플러그인](https://code.claude.com/docs/en/discover-plugins#code-intelligence)을 설치하기 전엔 비활성. 플러그인은 언어 서버 설정만 번들하고 서버 바이너리는 별도 설치 ([[11 플러그인]]).

### Monitor 도구

> Claude Code v2.1.98 이상 필요.

백그라운드에서 무언가를 감시하고 변화 시 대화를 멈추지 않고 반응한다. 로그 tail+에러 플래그, PR/CI 폴링+상태 변화 보고, 디렉터리 변경 감시, 장기 스크립트 출력 추적 등. Claude가 작은 스크립트를 짜 백그라운드 실행하고 출력 라인을 도착 순으로 받는다. 중단은 Claude에게 취소 요청하거나 세션 종료.

- Bash와 **동일한 권한 규칙**(allow/deny 패턴 공유).
- Bedrock/Vertex/Foundry 불가. `DISABLE_TELEMETRY` 또는 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 설정 시에도 불가.
- 플러그인은 활성화 시 자동 시작하는 monitor를 선언할 수 있다 ([[11 플러그인]]).

### NotebookEdit 도구 동작

Jupyter 노트북을 `cell_id`로 셀 단위 수정. Edit처럼 전역 문자열 치환을 하지 않는다.

| 모드 | 동작 |
| :--- | :--- |
| `replace` | 셀 소스 덮어쓰기 (기본) |
| `insert` | 타겟 뒤에 새 셀. `cell_id` 없으면 맨 앞. `cell_type`(`code`/`markdown`) 필요 |
| `delete` | 타겟 셀 제거 |

권한 규칙은 `Edit(...)` 경로 포맷 사용(예: `Edit(notebooks/**)`).

### PowerShell 도구

Windows에서 Git Bash 경유 대신 PowerShell로 실행. Git Bash 없는 Windows는 자동 활성, 있는 Windows는 점진 롤아웃, Linux/macOS/WSL은 opt-in.

```json
{
  "env": {
    "CLAUDE_CODE_USE_POWERSHELL_TOOL": "1"
  }
}
```

- Windows에서 롤아웃 opt-out: `0`. Linux/macOS/WSL은 PowerShell 7+ (`pwsh`)가 `PATH`에 있어야 함.
- Windows는 `pwsh.exe`(7+) 자동 감지, `powershell.exe`(5.1) fallback. 활성 시 PowerShell이 기본 셸, Git Bash 있으면 Bash 도구는 POSIX 스크립트용으로 유지.
- `-ExecutionPolicy Bypass`를 **프로세스 범위로만** 적용 → 기본 Windows에서 `.ps1`·모듈 임포트 동작. Group Policy `MachinePolicy`/`UserPolicy`는 미오버라이드(엔터프라이즈 잠금 유지). 머신 정책 존중: `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1`.

셸 선택 설정 3가지:

| 설정 | 위치 | 효과 |
| :--- | :--- | :--- |
| `"defaultShell": "powershell"` | settings.json | 인터랙티브 `!` 명령을 PowerShell로. 도구 활성 필요 |
| `"shell": "powershell"` | 개별 command 훅 | 해당 훅을 PowerShell로. 도구 활성과 무관(훅이 직접 spawn) |
| `shell: powershell` | 스킬 frontmatter | `` !`command` `` 블록을 PowerShell로. 도구 활성 필요 |

프리뷰 제한: PowerShell 프로필 미로드, Windows에서 샌드박싱 미지원.

### Read 도구 동작

파일 경로를 받아 라인 번호와 함께 내용 반환(항상 절대 경로 권장). 전체 읽기가 토큰 한도 초과 시 첫 페이지 + `PARTIAL view` 안내(`offset`/`limit`로 더 읽는 법). 명시적 `offset`/`limit`로도 초과하면 에러.

지원 파일 타입:
- **이미지**(PNG/JPG 등): raw 바이트가 아닌 시각 콘텐츠로 반환. 큰 이미지는 모델 한도에 맞게 리사이즈·재압축됨 → 픽셀 디테일이 필요하면 ImageMagick 등으로 먼저 crop 요청.
- **PDF**: 10페이지 이하는 통째로. 초과 시 `pages` 파라미터(`"1-5"`)로 한 번에 최대 20페이지.
- **Jupyter 노트북**: `.ipynb`는 모든 셀 + 출력(코드/마크다운/시각화) 반환.

디렉터리는 읽지 못함 → Bash로 `ls`.

### WebFetch 도구 동작

URL + 추출 프롬프트를 받아 페이지를 가져오고, HTML이면 Markdown으로 변환한 뒤, 작고 빠른 모델로 프롬프트를 돌려 그 **답변**을 Claude에 전달한다(원본 페이지 X). 변환 단계는 설정 불가.

> [!warning] 설계상 lossy
> 추출 프롬프트가 도달 내용을 결정한다. "페이지에 언급 없음" 결과가 사실은 프롬프트가 안 물어본 것일 수 있다. 더 구체적 프롬프트로 재요청하거나 Bash `curl`로 미가공 페이지를 받는다.

- HTTP URL은 HTTPS로 자동 업그레이드.
- 큰 페이지는 고정 문자 한도로 truncate.
- 응답은 15분 캐시.
- 다른 호스트로 리다이렉트 시 따라가지 않고 원본/타겟 URL을 알려주는 텍스트 반환 → Claude가 2차 호출.

default·`acceptEdits` 모드에서 새 도메인 첫 접근 시 프롬프트(미리 승인된 문서 도메인 제외). 사전 허용: `WebFetch(domain:example.com)`. `auto`·`bypassPermissions` 모드는 프롬프트 건너뜀 ([[05 권한]]). `deny`/`ask`/`allow`의 명시적 `WebFetch(domain:...)`는 사전승인셋보다 우선. `User-Agent`는 `Claude-User`로 시작, `Accept`는 Markdown 선호. [[18 보안과 샌드박스]] 네트워크 규칙은 별도라 sandboxed 프로세스 도달엔 별도 sandbox 규칙 필요.

### WebSearch 도구 동작

Anthropic 웹 검색 백엔드에 쿼리해 제목+URL 반환(페이지는 안 가져옴 → WebFetch로 후속). 호출당 최대 8회 백엔드 검색으로 내부 정제. `allowed_domains`로 특정 호스트만, `blocked_domains`로 제외(둘은 한 호출에 병용 불가). 백엔드 비설정. 다른 프로바이더로 검색하려면 검색 도구를 노출하는 [[10 MCP]] 서버 추가. 권한 규칙은 specifier 없이 bare `WebSearch`만.

> 가용성: Claude API·Microsoft Foundry 가능. Google Vertex AI는 Claude 4 모델(Opus/Sonnet/Haiku)에서 동작. Amazon Bedrock은 서버사이드 웹 검색 미노출 ([[17 클라우드 프로바이더]]).

### Write 도구 동작

전체 내용으로 새 파일 생성 또는 기존 파일 덮어쓰기(append/merge 안 함). 기존 경로면 현재 대화에서 한 번 이상 읽었어야 덮어쓰기 가능(미읽음 기존 파일 Write는 에러). 새 파일엔 미적용. Bash로 본 것도 충족(Edit과 동일 규칙). 부분 변경은 Write 대신 Edit.

## 슬래시 커맨드 전체 목록 (commands)

세션 안에서 `/`를 입력하면 사용 가능한 모든 커맨드가 뜨고, `/` 뒤에 글자를 치면 필터링된다. 커맨드는 **메시지 시작**에서만 인식되며, 이름 뒤 텍스트는 인자로 전달된다. `<arg>`는 필수, `[arg]`는 선택 인자다.

표기 구분:
- **[Skill]**: 번들 [스킬](https://code.claude.com/docs/en/skills#bundled-skills). 직접 작성하는 스킬처럼 동작하며 Claude가 관련 시 자동 호출 가능 ([[07 스킬]]).
- **[Workflow]**: 번들 [동적 워크플로우](https://code.claude.com/docs/en/workflows#bundled-workflows). 작업을 다수 서브에이전트로 팬아웃하고 백그라운드 실행.

> [!note] 모든 커맨드가 모두에게 보이지 않는다
> 플랫폼·플랜·환경에 따라 가용성이 다르다. 예: `/desktop`은 macOS/Windows + Claude 구독 시에만, `/upgrade`는 Pro/Max에서만 표시.

### 전형적 워크플로우별 커맨드

- **레포 첫 세션**: `/init`로 `CLAUDE.md` 시작 → `/memory`로 다듬기. `/mcp`·`/agents`로 서버/서브에이전트 셋업, `/permissions`로 승인 규칙.
- **작업 중**: `/plan` plan mode 진입, `/model`·`/effort` 추론량 조절, 대화가 길어지면 `/context`로 윈도우 점검·`/compact`로 요약, `/btw`로 히스토리를 오염시키지 않는 곁다리 질문.
- **병렬 실행**: `/agents`(서브에이전트 매니저), `/tasks`(백그라운드 작업 목록), `/background`(세션 전체를 백그라운드 에이전트로), `/batch`(코드베이스 전반 변경을 worktree별 분해).
- **출하 전**: `/diff`(변경 확인), `/code-review`(버그·정리, `--fix` 적용), `/review`·`/security-review`(깊은 read-only), `/code-review ultra`(클라우드 멀티에이전트 리뷰).
- **세션 간**: `/clear`(프로젝트 메모리 유지하며 새 작업), `/resume`·`/branch`(복귀/포크), `/teleport`(웹 세션을 터미널로), `/remote-control`(다른 기기에서 로컬 세션 이어가기).
- **문제 발생**: `/rewind`(체크포인트 롤백), `/doctor`·`/debug`(진단), `/feedback`(세션 컨텍스트 첨부 버그 리포트).
