# 백엔드 → AI 인프라 엔지니어 로드맵

허브: [[AI-Infra]]

대상: 6년차 백엔드, 클라우드/인프라 중급(Docker·CI/CD·일부 IaC), 목표 = LLM 서빙 + MLOps + 플랫폼/K8s, 주력 AWS, 사이드(홈랩·스팟 GPU) 학습.

## 전이 자산 vs 새로 채울 것
| 그대로 전이 (~80%) | 새로 얇게 얹기 |
|---|---|
| K8s·Docker·CI/CD → GitOps·GPU 스케줄링 | GPU 물리(VRAM·HBM·CUDA, OOM) |
| API·p99·로드밸런싱 → LLM 서빙 ([[01 LLM 서빙과 추론]]) | 추론 엔진(vLLM·SGLang) |
| Prometheus·Grafana·SRE → GenAI 관측성 | KServe·KEDA·Kueue·Karpenter ([[02 쿠버네티스 GPU 오케스트레이션]]) |
| 아티팩트·릴리스·롤백 → MLOps ([[03 MLOps 파이프라인과 관측성]]) | MLflow·Flyte·드리프트 |
| 클라우드 비용 감각 → FinOps | 토큰 경제학($/1M tokens·스팟 70-90%) |

## 4단계 로드맵 (18~36개월)
| 단계 | 목표 | 핵심 산출물 | 자격증 |
|---|---|---|---|
| **0–3개월** | 전이자산 매핑 + GPU 점화 | 스팟 GPU에 vLLM 8B 서빙 + k6 부하로 throughput·TTFT·p99 측정 GitHub 리포 + 8개 도메인 갭 자가진단 | — |
| **3–9개월** | 집중 3영역 깊이 | EKS+Karpenter(스팟) KServe/vLLM + KEDA + DCGM, 전부 Terraform IaC. vLLM vs SGLang 벤치 블로그 | **CKA** |
| **9–18개월** | 캡스톤 완성 | 위 + MLflow 레지스트리 + GitHub Actions CI/CD(register→canary→promote) + 토큰당 비용 대시보드 + Evidently 드리프트 = 면접에서 끝까지 설명 가능한 단일 시스템 | **AWS MLA-C01** |
| **18–36개월** | 차별화·시그널 | OSS 기여(KServe/vLLM PR) + 분산추론 또는 K8s Operator(Go) + 블로그/발표 → 이직·전환 | — |

## 체크리스트
- [ ] 8개 도메인 역량 갭 자가진단 표 작성
- [ ] 스팟 GPU 첫 vLLM 서빙 + 부하테스트 리포 (0-3개월)
- [ ] vLLM vs SGLang 벤치 블로그 1편
- [ ] EKS+Karpenter+KServe Terraform IaC 리포
- [ ] CKA 취득 (3-9개월)
- [ ] 캡스톤: 자체 LLM 추론 플랫폼(IaC+CI/CD+관측성+비용) (9-18개월)
- [ ] AWS MLA-C01 취득
- [ ] OSS PR 1건 머지 (18-36개월)
- [ ] 기술 블로그 시리즈 / 밋업 발표
- [ ] 목표 JD 5건 역량 매핑 → 타깃 지원

## 핵심 전략
1. **이중 트랙(AWS)**: Bedrock/SageMaker(매니지드, 빠른 결과) ‖ EKS+오픈소스(깊이). 메인 실습은 `awslabs/ai-on-eks`.
2. **단일 캡스톤 > 자격증**: 흩어진 토이 N개보다 "프로덕션처럼 보이는 단일 시스템 1개".
3. **백엔드 자산을 ML 언어로 번역**: 분산시스템·온콜·CI/CD를 면접에서 전면에. 신입 ML 지원자 대비 유일·결정적 우위.
4. **GPU 없이 시작**: CPU/작은 모델 → 스팟 GPU(시간당) → Modal/RunPod 서버리스. 첫 주에 스팟+자동종료+예산 알람.

## 함정 (시간 낭비 방지)
- 자격증 컬렉터 → CKA + MLA-C01 **2개로 끝** (구 ML Specialty는 2026-03-31 은퇴).
- 튜토리얼 지옥 → 강의 1개(Made With ML)로 제한, 80%는 손으로.
- 도구 수집가 → vLLM/Flyte 하나씩 깊게 (얕게 10개 < 깊게 1개).
- 모델 학습 과투자 → 너는 "서빙·운영"하는 사람. 파인튜닝/논문 재현 금지.
- GPU 비용 폭탄 → 스팟 + 자동 종료 + 예산 알람.
- 백엔드 자산 과소평가 → 기초에 시간 낭비 말 것, 전이 자산 위에 얇게.
