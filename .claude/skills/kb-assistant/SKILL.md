# KB Assistant — 의도 라우터

이 vault의 콘텐츠는 `Claude/` 지식 베이스(KB) 하나다 — Claude Code 공식 문서(code.claude.com/docs)를 한국어로 종합한 26개 노트(`01 시작하기` ~ `26 변경 이력과 용어집`)와 중심 허브 MOC `Claude/Claude.md`. 사용자의 자연어 의도를 아래 작업으로 연결하라. 사용자가 슬래시 커맨드를 직접 칠 필요 없이, 의도만으로 올바른 경로를 택한다.

## 의도 → 작업 매핑

| 사용자 의도 | 실행 | 판단 기준 |
|---|---|---|
| Claude Code **사용법 질문** ("~어떻게 해", "~기능 있어?", "전에 정리한 거") | `/kb-query` 절차 — MOC→노트로 답한다. **답이 여러 노트에 흩어져 다중 노트를 읽어야 하면** `kb-guide` 서브에이전트에 위임(읽기 전용 격리, 답+근거만 반환). | 한두 노트로 끝날 단순 조회면 메인에서 직접; 여러 노트 종합·원문 다량 읽기면 위임. |
| 공식 문서 **변경분 반영·KB 최신화** ("문서 최신화", "바뀐 거 반영") | `/kb-sync` 절차 — llms.txt 슬러그 diff로 변경분을 잡아 반영. **다중 노트 갱신·대량 fetch가 예상되면** `kb-updater` 서브에이전트에 위임(쓰기 격리, ①②만 수행, hot.md는 안 건드림). | 노트 1~2개 소규모 패치면 메인; 광범위 변경이면 위임. |
| KB **건강 점검** ("상태 점검", "끊긴 링크 있어?") | `/kb-lint` 절차 — 먼저 `python3 .claude/kb-lint.py`(기계적: 필수필드·날짜형식·끊긴 `[[링크]]`·빈 노트·코드펜스; 필요시 `--online`으로 공식 인덱스 대조), 이어 의미적 점검(MOC 누락·요약 drift·고아·모순). | 점검은 항상 기계 린트 먼저, 그 다음 LLM 의미 점검. |
| 외부 **자료를 KB에 박제** ("이 글/URL 정리해줘") | `/kb-ingest` 절차 — 자료를 캡처해 알맞은 노트에 통합하고 MOC·교차링크 반영. | 사용자가 외부 URL·문서·기사를 KB에 넣으려 할 때. |
| 방금 한 **대화·탐색을 박제** ("노트로 남겨줘", "방금 알아낸 거 정리") | `/kb-save` 절차 — 이번 세션의 통찰을 노트로 남긴다. | 새 외부 자료가 아니라 지금 세션에서 나온 결과를 고정할 때. |

## 원칙
- KB는 사실 레퍼런스다. 명령어·플래그·설정 키·환경변수는 원문 영어 그대로, 맥락·설명은 한국어. 노트 frontmatter는 3필드(`title`/`updated`/`sources`).
- **갱신 의무**: KB 노트를 만들거나 바꾸면 `.claude/rules/vault-rules.md`의 "Update duty (CANONICAL — 3 steps)"를 수행한다(정본은 vault-rules.md — 여기서 단계를 재명세하지 않는다).
- **서브에이전트 위임 시 hot.md 단계는 메인 세션 몫**: `kb-guide`·`kb-updater`는 격리 컨텍스트에서 노트 측 단계(`updated`·MOC)까지만(읽기 전용 에이전트는 그것도 없음) 하고, hot.md는 에이전트가 반환한 요약을 받아 메인이 갱신한다.
- 무거운 작업(여러 노트 동시 읽기/쓰기, 대량 fetch)은 서브에이전트로 격리해 메인 컨텍스트를 보호한다 — 이것이 비용·정확도 레버다.
