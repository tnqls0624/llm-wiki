# KB Save: 대화를 KB에 남긴다

저장 힌트: `$ARGUMENTS`

`/kb-ingest`가 외부 자료를 박제한다면, `/kb-save`는 그 대화판이다 — **이번 세션에서 만들어진 통찰**을 입력으로 받아 검색 없이 KB에 고정한다. 채팅에만 남아 휘발될 결론을 지식으로 굳히는 명령. KB 노트 형식은 frontmatter 3필드(`title`/`updated`/`sources`) + 한국어 본문 + `[[위키링크]]` + 말미 `## 원본 문서`를 따른다.

## 1. 식별
- 이번 세션에서 **저장 가치가 있는 통찰·결론·비교·결정**을 추린다. `$ARGUMENTS`가 있으면 그 범위/제목 힌트를 우선한다.
- 단순 Q&A(노트에 이미 있는 내용을 조회한 것)였다면 **저장하지 않고 그 이유를 밝히고 종료한다** — 노이즈 노트를 만들지 않는다.

## 2. 종합 → 노트 위치
- Claude Code 관련 통찰이면 → `Claude/`의 주제가 맞는 기존 노트를 보강하거나, 분량이 충분한 새 종합이면 번호를 이어서 새 `Claude/NN 제목.md`를 만든다.
- 다른 주제면 → `/kb-ingest`와 동일하게 새 주제 디렉토리(`<주제>/` + `<주제>/<주제>.md` MOC)에 노트를 만든다.
- 대화가 근거이므로 frontmatter `sources`에는 참조한 KB 노트나 외부 출처를 적고, 대화 자체가 유일한 근거면 그렇게 명시한다(예: `sources: [conversation-2026-06-07]`).

## 3. 작성·교차참조
- 본문은 한국어로 통찰을 정리한다. 추론·미확정 부분은 불확실성을 표시한다.
- 관련 KB 노트로 `[[위키링크]]` 교차참조를 건다(파일명만). 인용한 노트 쪽 역링크도 고려한다. 기존 내용과 모순되면 삭제하지 말고 `> [!warning] 모순` 콜아웃으로 표시. 말미에 `## 원본 문서`(대화 기반이면 그렇게 표기).

## 4. 갱신 의무
- `.claude/rules/vault-rules.md`의 "Update duty (CANONICAL — 3 steps)"를 수행한다(정본 참조 — 단계·수치는 여기서 재명세하지 않는다). MOC는 만진 디렉토리의 허브(`Claude/Claude.md` 또는 새 주제의 `<주제>/<주제>.md`)를 갱신한다.
- 새 주제 디렉토리를 만들었다면 `hot.md`의 `## Vault state`에도 반영한다.
