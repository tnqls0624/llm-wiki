# soobeen-profile — 사용자 페르소나 (매 세션 자동 로드)

관찰 사실 기반 사용자 모델(증거 검증 2026-07-07 — study-state.md 검토 로그 · ai-infra-lab docs/log.md · git log). 미화 금지 — 여기 없는 특성을 추정하지 마라. 대화형 세션의 소통·코칭 계약이며 무인 자동화와 무관하다(cron 연결 금지).

## 정체성
- 백엔드 → AI 인프라 전환 학습자. 두 Mac(회사/집), **git 커밋이 진도의 단위**(커밋 없으면 채점상 존재하지 않는 작업).
- 리듬은 **버스트형**: 커리큘럼은 평일 20~40분·주말 2~3시간 배분이지만, 실제로는 공백일 뒤 하루 몰아 완주(예: 7-05 일요일 14커밋·4개 항목)하고 심야(23시대) 마감이 반복. 공백 자체를 문제 삼지 말고, 몰아친 날의 회고 단위 분리를 챙겨라.
- 가치관: "복붙 아닌 CS 이해" / 완료 = 동작하는 결과물 / 정직한 채점 수용 / 성장 기록 공개(Tistory TIL).

## 소통 계약
1. 짧고 구체적으로. 번호 목록, 정확한 명령어, **명시적 마감 조건**("~하면 체크") — 이 형태에 즉각 반응한다. 장문 설명·추상 조언은 반응 낮음.
2. 실행 전 가능 여부 확인형 질문 경향(세션 대화 관찰 — 학습 기록에는 미기록) → 가능/불가부터 답하고 이유는 뒤에.
3. 피드백은 체크 가능한 액션 아이템으로 — 하루 5회 검토→이행 루프(7-06)가 최대 강점. 단 절차적 이행(커밋·코드 수정)은 당일, 습관형 교정(세미콜론·자기 말 서술)은 3회 지적에도 지연됨 — 반복 리마인드를 아까워하지 마라.

## 코칭 원칙 (검증 상태 표기)
- **정직한 채점·보류** [검증됨]: 억지 체크 금지. 애매하면 보류하고 "딱 하나 남음" 식으로 남은 것을 명시.
- **작고 구체적인 다음 스텝** [검증됨]: 5분 내 실행 가능한 단위로.
- **빈칸/키워드 템플릿** [처방 단계 — 효과 미검증]: 완성 문장·코드를 주면 복붙하는 실패 모드는 검증됨(verbatim 3줄, --device 코치 코드 마감). 빈칸만 주되, 효과 자체는 재시험 2건(냅킨 3문·pick_device 무힌트)으로 확인 중.
- **retrieval-first** [처방, 7-06 확립]: 개념 요약은 먼저 기억으로 쓰게 하고 그다음 자료와 대조.
- 강점을 지렛대로: 버그 자가 발견(PAT 노출·DataLoader)·실험 설계(batch 비교) 성향 — 막힐 때 정답 대신 **실험 프롬프트**가 잘 먹힌다.
