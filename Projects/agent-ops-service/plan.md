# agent-ops-service — 기획 (plan)

> 서비스+콘텐츠 병행 트랙 (2026-06-08, GEO 보류 후 전환). 두 회의 학습 반영: 청중 0이 공통 병목 → 콘텐츠로 해결; founder-fit은 무료 복제 안 되는 서비스에 있음. **빌드 없음 → 실행=검증.**

## 한 줄
Claude Code 에이전트·무인 자동화를 **안전하게 운영**하도록 셋업·진단해주는 1:1 서비스 + 그 노하우를 build-in-public 콘텐츠로 풀어 청중·신뢰를 쌓는다. 콘텐츠=마케팅(무료 채널), 서비스=수익.

## 왜 이게 founder-fit인가 (무료로 복제 안 됨)
이 vault가 살아있는 PoC다 — `automation-safety-rules`(최소권한·신뢰경계·심층방어), collect↔review 분리, 무인 cron 사후 `git checkout` 가드, fetch-guarded push, 계약 테스트(`test_mechanisms.py`), context layering(hot.md→MOC→노트)을 **직접 설계·운영 중**. 시중 강의·무료 OSS가 말로만 하는 "에이전트 안전 운영"을 코드로 굴려본 경험이 곧 해자. (GEO처럼 무료로 깔리는 도구가 아님.)

## 서비스 정의
- **대상**: Claude Code/에이전트를 cron·헤드리스로 돌리(려)는 **한국 1인 개발자·소규모 팀·작은 AI 에이전시**. (엔터프라이즈 아님 — 1인이 신뢰 못 사는 영역 회피.)
- **진단 체크리스트**(automation-safety-rules 기반, 12항목): 권한 allowlist 스코프 · 무인 범위 밖 변경 가드 · collect↔review 분리 · 시크릿 스캔 · 계약 테스트 유무 · fetch-guarded push · context layering · 비용/모델 티어 · 프롬프트 인젝션 방어 등.
- **형태**: 무료 30분 진단(리드) → 유료 정비·셋업 고정가(₩30만~80만) 또는 월 리테이너. 산출물 = 12항목 진단 리포트 + 가드 설치.
- **확장성 보완**(비판가 우려): 진단을 표준 체크리스트 + (추후)자동 스캐너로 반복가능하게 만들어 컨설팅의 시간노동을 준-제품화. 리드는 콘텐츠가 자동 공급.

## 콘텐츠 정의 (청중 = 모든 것의 전제)
- **주제**: "AI 직원들과 1인 회사 굴리기 / 무인 에이전트 안전하게 운영하기".
- **소스(이미 존재)**: 이 vault의 실제 사례 — git 자동커밋 로그, automation-safety 적용, collect↔review, 계약 테스트, **이번 사업 회의 2건의 "검증으로 죽은 길을 돈 0으로 거른" 스토리**. 생산 비용 ≈ 0(일하면서 생김).
- **채널(개발자, 무료)**: GeekNews · 디스콰이엇 · 요즘IT · OKKY · 링크드인. (아이보스는 마케터용이라 제외.)

## 첫 1개 실행 (비용 0, 실행=검증)
1. **build-in-public 1호 글**: "AI 직원 7명 + 비판가로 1인 창업 회의를 돌렸더니, 2주 가짜문을 만들기도 전에 아이디어가 죽었다" — 실제 스토리(이 vault). 채널 1곳(디스콰이엇 또는 GeekNews).
2. **무료 진단 오퍼**: 글 말미 CTA — "Claude Code를 무인으로 돌리는데 범위 밖 커밋·시크릿 유출·폭주가 걱정되면 무료 30분 안전 진단해드립니다."
- **측정**: 글 반응(저장·댓글·DM) + **무료 진단 신청 건수**(=서비스 수요 직접 신호).
- **합격선**: 2주 내 무료 진단 신청 ≥3건 또는 의미있는 인게이지먼트(DM/"더 보고싶다") ≥5건. 신청 0이면 메시지·채널·타깃 재조정.

## 실행 플랜 (2026-06-08 실행 회의 확정)
비판가 우선순위(부트스트랩 4대 잣대):

2. **[9] agentsafe-doctor 스캐너 '치명 5항목'** 1~2일 빌드(보유 `scrub-secrets.py` 재사용) → 무료진단을 '대화'가 아닌 '네 `.claude/` 스캔 리포트 리뷰'로 표준화. 항목은 Routines native 가드와 **안 겹치는 것만**: ①비용/토큰 폭주 가드 ②hook recursion 타임아웃·depth ③시크릿 스캔 ④외부텍스트 인젝션 중립화 ⑤계약 테스트(positive control) 존재. 시간노동 30분→2분(확장성 해소).
3. **[8] build-in-public 3채널 재활용**(디스콰이엇 풀→GeekNews 요약+링크→링크드인 한문단) 주 1회. 프레임 전환: '내 cron이 안전하다'가 아니라 **'공식 Routines가 자율로 돌아도 비용폭주·recursion·인젝션은 네 책임 — 내가 깨먹고 막은 실제 코드'**.
4. **[8] 패키징 0+1단계만 먼저**: 0단계 무료 30분 진단(주 3건 상한) + 1단계 Audit Pack **₩15만**(스캐너 전체 리포트, 후속 정비 시 전액 차감). ₩35~70만 정비는 진단팩 1건 팔린 뒤 빌드(수요검증 후).
