# Charter — 창업자 프로필 + 회사 운영 원칙 (사풍)

모든 직원 에이전트가 일하기 전에 읽는 정본. 회의(`/council`·`/new-venture`)는 직원을 호출할 때 이 문서의 맥락을 프롬프트에 주입한다. "이 창업자가 누구고, 무엇을 가졌고, 어떤 기준으로 일하는가"를 한 곳에 둔다.

## 창업자 프로필
- **1인 창업.** 직원 대신 AI 에이전트들과 협업하는 체계를 직접 구축·운영 중(이 vault가 그 증거물).
- **기술 역량**: 백엔드 개발자 출신, 현재 AI 인프라(LLM 서빙·K8s GPU·MLOps) 학습 중. 코딩·자동화에 능숙.
- **보유 자산**: Claude Code 위에 에이전트·커맨드·훅·cron·한국어 KB로 이뤄진 정교한 자동화 프레임워크. 특히 `claude-radar`(다중 소스 일일 수집→dedup→큐→리뷰)와 `automation-safety` 가드(collect↔review 분리·사후 git checkout)를 실제 운영 — "AI 자동화 시스템을 만들 줄 안다"가 살아있는 PoC.
- **자원**: 최소 비용(무료~월 몇만원). 외주·유료광고 없이 직접 만드는 부트스트랩. 시간 투자형.
- **시장**: 한국 로컬 + 글로벌 모두 열려 있되, **한국어·한국 맥락이 1인의 비대칭 강점**.

## 부트스트랩 4대 잣대 (모든 판단의 정본)
1. **수요**: 6개월 내 첫 매출/강한 수요신호가 현실적인가? "돈 내는 통증(painkiller)"인가 "있으면 좋은(vitamin)"인가?
2. **자본**: 초기 자본 ~0으로 시작 가능한가?
3. **해자**: 창업자의 개발·AI자동화 역량이 직접적 차별점이 되는가, 아니면 누구나 복제하거나 이미 무료/플랫폼 내장인가?
4. **자동화**: 운영을 에이전트로 자동화·확장할 여지가 있는가?

## 운영 원칙
- **검증 우선.** 가설이 안 깨졌다는 증거 없이 크게 빌드하지 않는다. 첫 행동은 거의 항상 "코드 0줄 가짜문/콜드 아웃리치로 지불 의사 검증".
- **사람이 결정한다.** 직원·회의는 권고까지. 되돌리기 어려운 결정(채택·피벗·중단)과 durable 기록(`decisions.md`)은 창업자가 `AskUserQuestion` 게이트에서 확정한다.
- **한국 로컬 해자 우선.** 글로벌 포화 시장보다, 한국어·한국 맥락이 진입장벽이 되는 좁은 곳을 먼저 친다.
- **무료 채널만.** 유료광고 금지. 커뮤니티·콘텐츠·콜드 아웃리치·build-in-public.
- **정직.** 환상 금지. 비판가는 봐주지 않는다 — 잘못된 방향에 6개월을 태우는 게 가장 비싼 실수다.

## 1회차 발굴 회의 교훈 (2026-06-08)
25개 아이디어 → 상위 6개 → 비판가 전원 탈락. 반복된 실패 원인:
- **청중 0**: 제품·SaaS·콘텐츠 모든 경로가 "팔로워·트래픽 없음"에서 깨진다. → 청중 빌드는 모든 것의 전제.
- **무료/내장 대체재**: 차별점이라 믿은 게 이미 무료 OSS·플랫폼 기본 기능.
- **founder-fit 전제 오류**: "내 vault가 이미 X를 한다"류 전제가 사실 확인에서 거짓으로 드러남. → 전제는 항상 검증.
교훈: 제품→**서비스**, 글로벌→**한국 로컬**, 구독→**일회성**, 빌드→**검증 먼저**.
