# geo-citation-report — 기획 (plan)

> 첫 검증 트랙. 2026-06-08 발굴 회의 + 비판가 검증 처방 반영. **빌드 전 2주 지불의사 검증부터.**

## 문제정의
한국 B2B SaaS·중견 브랜드는 고객이 점점 ChatGPT·Perplexity 같은 AI에게 "어떤 제품이 좋아?"를 묻는데, 자기 브랜드가 그 답변에 인용되는지 모른다. 검색(SEO)은 추적 도구가 많지만, **AI 답변 인용(GEO)**은 한국 시장에서 가시성 도구가 비어 있다.

## 가설 (검증 가능 문장)
> 우리는 **[한국 B2B SaaS·중견 브랜드의 마케팅 담당자]**가 **[자기 브랜드가 AI 답변에 인용되는지 모른다는 불안]** 때문에 **[ChatGPT·Perplexity 인용 진단 리포트]**에 **[일회성 ₩29,000~]**를 낼 것이라 믿는다.
> 이게 틀렸다면 → **무료 진단 리포트를 받고도 단건 유료 결제 시도가 10% 미만**이면 안다.

## 비판가 처방 (반드시 반영)
- ❌ 구독 SaaS → ✅ **일회성 유료 진단 리포트** (SMB는 지속 모니터링 아닌 단건 감사를 산다).
- ❌ 네이버 추적 → ✅ **네이버 제외** (robots.txt AI크롤러 차단 + 한국 판례상 법적 리스크). ChatGPT·Perplexity·Gemini 공식 API/Apify만.
- ❌ 자영업 타깃 → ✅ **한국 B2B SaaS·중견 브랜드** (자영업은 가격민감·해석 시간 없음 = 틀린 고객).
- ⚠ 측정 노이즈: 같은 브랜드 진단을 시차 두고 2회 돌려 결과 흔들림을 직접 확인(흔들리면 신뢰 기반이 무너지므로 빌드 전에 안다).

## founder-fit (해자)
- claude-radar의 수집→dedup→큐 파이프라인 = 진단 데이터 엔진과 동일 구조. 수백 쿼리를 매일 LLM에 던져 파싱·점수화하는 건 개발자만 하고, 이미 만들어 봤다.
- 한국어 + 한국 B2B 맥락 = 글로벌 경쟁자 대비 비대칭.

## 6개월 마일스톤 (첫-수요-신호까지)
| 주차 | 목표 | 성공지표 |
|---|---|---|
| W1~2 | **가짜문 검증** (아래 "다음 1개 실험") | 단건 유료 결제 시도 ≥10% |
| W3~4 | 합격 시: 진단 자동화 MVP(빌더) — 한국 브랜드 1곳 쿼리→파싱→1p 리포트 | 30분 내 1개 리포트 자동 생성 |
| W5~8 | 유료 진단 5건 판매 (콜드 아웃리치 + 케이스) | 첫 매출 ₩100k+ |
| ~6개월 | 단건 반복 수요 확인 시에만 셀프서브/화이트라벨 검토 | 월 진단 의뢰 10건+ |

## 다음 1개 실험 (지금 당장 — 코드 0줄)
1. 한국 B2B SaaS·중견 브랜드 **15~30곳**을 손으로 고른다.
2. 각 브랜드를 ChatGPT·Perplexity에 실제 질의해 인용 여부를 **수동 캡처**(Claude로 파싱·작성 자동화 = founder-fit) → **1페이지 진단 리포트** 작성.
3. 콜드 이메일/링크드인으로 무료 리포트 발송 + 말미에 **단 하나의 CTA**: "더 깊은 리포트 ₩29,000" + 가짜 결제 링크(Stripe payment link).
4. **측정 지표**: 리포트 열람률이 아니라 **결제 시도/카드 등록 전환율**. 동시에 "매달 받겠나"(구독 의향)도 명시 질문.
5. **합격선**: 15~30곳 중 단건 유료 시도 **3건(10%)+** → MVP 진행. 0~1건 → 폐기 또는 타깃 피벗.
